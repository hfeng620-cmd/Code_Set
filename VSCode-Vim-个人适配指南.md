# VS Code + Vim 个人适配指南

适用对象：当前这台设备上的 VS Code 和 VSCodeVim 配置。

这不是标准 Vim 教程，也不是 VS Code 官方快捷键大全。它只讲你现在这套配置里，最值得记住、最常用、最不容易搞混的内容。

## 先记住一个核心原则

你现在是双模式使用：

- 插入模式：更像普通 VS Code 编辑器，优先使用 VS Code 原生快捷键。
- 普通模式：更像 Vim，移动、跳转、窗口和部分 `Ctrl` 键优先使用 Vim 行为。

最重要的切换方式：

- 进入插入模式：按 `i`
- 从插入模式回到普通模式：按 `Esc`
- 你还额外配置了：插入模式按 `jj` 也能退出到普通模式

如果你一时不知道自己在哪个模式，先按一次 `Esc`，回到普通模式再操作，通常最稳。

## 你现在的自定义键位

这些是你已经从 Neovim/个人习惯迁过来的，优先记这些。

### Leader 键

你的 `leader` 是空格，也就是：

- `Space`

所以像 `<leader>ff`，实际就是：

- `Space f f`

### 文件和搜索

- `Space f f`：快速打开文件
- `Space f g`：全局搜索
- `Space f b`：显示已打开的编辑器/缓冲区列表

### 侧边栏和分屏

- `Space e`：切换侧边栏
- `Space s v`：向右分屏
- `Space s h`：向下分屏

### 保存和关闭

- `Space w`：保存文件
- `Space q`：关闭当前编辑器

### 终端

- `Space t t`：切换终端

### 代码跳转和 LSP

- `g d`：转到定义
- `g D`：转到声明
- `g r`：查找引用
- `g i`：转到实现
- `K`：查看悬停说明
- `Space r n`：重命名符号
- `Space c a`：快速修复 / 代码操作
- `Space f m`：格式化文档

这些已经很够用了。只把这批记熟，你日常写代码就会顺很多。

## 你最常用的工作流

### 1. 打开项目后找文件

普通模式下：

- `Space f f` 找文件
- `Space f g` 全局搜索关键字
- `Space e` 打开或收起左侧栏

如果你想用 VS Code 原生方式：

- `Ctrl+P`：快速打开文件
- `Ctrl+Shift+F`：全局搜索
- `Ctrl+Shift+P`：命令面板

### 2. 编辑一段代码

推荐习惯：

1. 用普通模式移动到目标位置
2. 按 `i` 进入插入模式
3. 正常输入
4. 按 `Esc` 或 `jj` 回到普通模式

最基础的普通模式移动：

- `h` `j` `k` `l`：左下上右
- `w`：下一个单词开头
- `b`：上一个单词开头
- `0`：行首
- `$`：行尾
- `g g`：文件开头
- `Shift+g`：文件结尾

### 3. 查定义、改名字、修格式

普通模式下，优先用这组：

- `g d`：看定义
- `g r`：看引用
- `g i`：看实现
- `K`：看说明
- `Space r n`：重命名
- `Space c a`：修复建议
- `Space f m`：格式化

这套已经覆盖了大部分代码阅读和修改场景。

## 你当前最容易混淆的地方

### 插入模式和普通模式不是一回事

你现在已经把一批常用 `Ctrl` 键改成了这种规则：

- 插入模式：优先走 VS Code 原生快捷键
- 普通模式：优先走 Vim 插件行为

这意味着同一个按键，在不同模式下可能完全不是一件事。

### 下面这些键，插入模式和普通模式差异比较大

- `Ctrl+S`
- `Ctrl+Z`
- `Ctrl+X`
- `Ctrl+V`
- `Ctrl+D`
- `Ctrl+U`
- `Ctrl+R`
- `Ctrl+K`
- `Ctrl+N`
- `Ctrl+P`

建议这样记：

- 想“编辑器功能”，先进入插入模式
- 想“Vim 导航/命令”，先回普通模式

## 什么时候用 VS Code 原生，什么时候用 Vim

可以用这个简单判断：

- 和“输入、保存、复制粘贴、打开面板”有关：优先想 VS Code
- 和“移动、跳转、查定义、窗口操作”有关：优先想 Vim

换句话说：

- 插入模式：像普通编辑器
- 普通模式：像代码导航器

## 多行编辑专用

这一节只讲你最容易马上用上的多行操作。

### 复制

- `yy`：复制当前行
- `5yy`：从当前行开始，复制共 5 行
- `V` 然后 `j` / `k` 再 `y`：按行选中多行后复制

例子：

- `V j j y`：选中当前行和下面两行，再复制
- `V 5k y`：选中当前行和上面 5 行，再复制

### 删除

- `dd`：删除当前行
- `5dd`：从当前行开始，删除共 5 行
- `V` 然后 `j` / `k` 再 `d`：按行选中多行后删除

### 粘贴

- `p`：粘贴到当前光标后面 / 当前行下方
- `P`：粘贴到当前光标前面 / 当前行上方

如果你复制的是整行，`p` 和 `P` 的体验可以理解成：

- `p`：贴到下面
- `P`：贴到上面

### 缩进

- `>>`：当前行右缩进一次
- `<<`：当前行左缩进一次
- `5>>`：当前行开始往下 5 行整体右缩进
- `5<<`：当前行开始往下 5 行整体左缩进

可视模式下也很常用：

- `V` 选中多行后按 `>`
- `V` 选中多行后按 `<`

### 块选择

- `Ctrl+V`：进入可视块模式

这个适合做竖着一列的编辑，比如多行前面一起加内容、删内容。

最常见的入门用法：

1. `Ctrl+V` 进入块选择
2. 用 `j` `k` 扩展到多行
3. 用 `l` `h` 调整列范围
4. 按 `d` 删除这一块，或者按 `I` 在块前插入

如果你是刚开始用，先把 `V` 的按行选择用熟，再碰 `Ctrl+V` 会轻松很多。

### 改动一整段文本

这几个也很常用：

- `D`：从光标位置删到行尾
- `C`：从光标位置改到行尾
- `d$`：删除到行尾
- `cc`：改整行
- `S`：替换整行内容

可以把它们理解成：

- `d`：删
- `c`：删掉后直接进入插入模式开始改

### 最值得先练的多行操作

先别全背，优先练这几个：

- `yy`
- `5yy`
- `dd`
- `5dd`
- `V` + `j/k` + `y`
- `V` + `j/k` + `d`
- `>`
- `<`
- `p`
- `P`

## 如果之后你还想继续优化

你下一步最值得考虑的，不是加更多快捷键，而是先把下面几个练熟：

- `Ctrl+D` / `Ctrl+U`
- `g d` / `g r` / `g i`
- `Space f f` / `Space f g`
- `Space r n` / `Space c a`

把这几组练熟以后，再决定要不要继续定制更多键位，效率会更高。
