## 2. 代理组层次结构

```
飞猪云 (select) ──── 顶级选择器
  │
  ├── Auto (fallback) ─── 自动故障转移
  │     ├── 测试 URL: http://www.gstatic.com/generate_204
  │     ├── 检测间隔: 7200 秒
  │     └── 包含全部 34 个节点
  │
  ├── YouTube     (select, 默认选飞猪云)
  ├── Disney      (select, 默认选飞猪云)
  ├── Hbomax      (select, 默认选飞猪云)
  ├── Netflix     (select, 默认选飞猪云)
  ├── Bahamut     (select, 默认选飞猪云)
  ├── Bilibili    (select, 有 DIRECT 选项)
  ├── Spotify     (select, 有 DIRECT 选项)
  ├── Steam       (select, 有 DIRECT 选项)
  ├── Telegram    (select, 默认选飞猪云)
  ├── Google      (select, 默认选飞猪云)
  ├── Microsoft   (select, 有 DIRECT 选项)
  ├── OpenAI      (select, 默认选飞猪云) ← Codex 走这里
  ├── Apple       (select, 有 DIRECT 选项)
  └── Final       (select, 有 DIRECT 选项) ← 所有未匹配流量
```

### 关键理解

1. **飞猪云** 是根选择器。改飞猪云的选中项，等于同时改了 12 个服务组的默认路由。
2. **Auto** 是 fallback 类型——它自动选择所有在线节点中延迟最低的那个。节点挂了会自动切换。
3. **OpenAI 组**没有 DIRECT 选项——意味着 OpenAI 流量强制走代理（因为国内直连连不上）。
4. **Bilibili/Steam/Spotify/Microsoft/Apple/Final** 有 DIRECT 选项——可以手动切换到直连。

### 代理组类型说明

| 类型 | 行为 |
|------|------|
| `select` | 手动选择一个节点/组，不自动切换 |
| `fallback` | 健康检查 + 自动选择最快的在线节点 |
| `url-test` | 持续测速，自动选最快的 |
| `load-balance` | 在节点间负载均衡 |
