## 2. PROCESS-NAME 规则（进程名匹配）

这些规则根据可执行文件名来路由流量。**这是最优先的匹配规则之一**。

```
行号   规则                              目标      说明
─────────────────────────────────────────────────────
988    PROCESS-NAME,claude              DIRECT    Claude Code → 直连（DeepSeek API）
989    PROCESS-NAME,opencode            DIRECT    OpenCode → 直连（DeepSeek API）
990    PROCESS-NAME,.opencode           DIRECT    OpenCode 子进程 → 直连
991    PROCESS-NAME,codex               飞猪云    Codex → 代理（OpenAI API 必须走代理）
```

**为什么 Claude 直连、Codex 走代理？**

| 工具 | 目标 API | 国内可直连？ | 路由 |
|------|----------|------------|------|
| Claude Code | `api.deepseek.com` / `glm.ai` | ✓ | DIRECT |
| Codex | `chatgpt.com` / `api.openai.com` | ✗ | OpenAI 组 |
| OpenCode | `api.deepseek.com` | ✓ | DIRECT |

### Codex 的流量路径

Codex 发起请求的完整链路：

```
codex 进程启动
  │
  ├── HTTP_PROXY=http://127.0.0.1:7897（shell 环境变量）
  │   或通过 TUN 透明代理
  │
  ├── 目标: chatgpt.com:443
  │
  ├── Clash 规则匹配:
  │   1. PROCESS-NAME,codex → 命中！→ 走 OpenAI 组
  │   2. （如果没有进程规则，DOMAIN-SUFFIX,chatgpt.com 也会命中 OpenAI）
  │
  ├── OpenAI 组选择: 飞猪云 → Auto → 最快代理节点
  │
  └── 实际出站: 通过 Trojan 节点 → chatgpt.com → Cloudflare → OpenAI 后端
```

## 4. 直接连接（DIRECT）规则

### 中国 AI 服务（直连）

```yaml
- DOMAIN-SUFFIX,deepseek.com,DIRECT
- DOMAIN-SUFFIX,deepseek.ai,DIRECT
- DOMAIN-SUFFIX,dashscope.aliyuncs.com,DIRECT   # 阿里百炼
- DOMAIN-SUFFIX,glm.ai,DIRECT                   # 智谱
- DOMAIN-SUFFIX,moonshot.cn,DIRECT              # Moonshot
```

### 中国站点（直连）

```yaml
- DOMAIN-SUFFIX,qq.com,DIRECT
- DOMAIN-SUFFIX,wechat.com,DIRECT
- DOMAIN-SUFFIX,tencent.com,DIRECT
- GEOSITE,cn,DIRECT          # 所有中国域名自动直连
- GEOIP,CN,DIRECT            # 所有中国 IP 自动直连
```

这是国内流量不浪费代理流量的关键——所有 `.cn` 域名和中国 IP 全部走直连。

## 5. MATCH 兜底规则（第 5006 行）

```yaml
- MATCH,Final
```

所有不匹配上述规则的流量，最终进入 **Final 组**。

Final 组默认选**飞猪云**（即走代理），也可以手动切换到 **DIRECT**。

**实际意义**：未知的国际流量默认走代理，国内流量已经在前面被 `GEOSITE,cn` 和 `GEOIP,CN` 截获直连了。
