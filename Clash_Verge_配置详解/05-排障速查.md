## 常见问题

### 1. 火狐打不开任何网站

**检查**：系统代理 mode 是否为 `manual`。
```bash
gsettings get org.gnome.system.proxy mode   # 应为 'manual'
```
如果不是：`gsettings set org.gnome.system.proxy mode 'manual'`

### 2. Codex 规则模式下不能用，全局模式可以

**原因**：规则文件里 `PROCESS-NAME,codex` 被错误指向了 `DIRECT`（直连）。
Codex 用 OpenAI API（国内被墙），直连必然超时。

**修复**：确认 `rbNfRMyQHAqC.yaml` 中是 `PROCESS-NAME,codex,飞猪云` 而非 DIRECT。

### 3. 国内网站走代理浪费流量

检查规则中是否有：
```yaml
- GEOSITE,cn,DIRECT
- GEOIP,CN,DIRECT
```
如果没有，说明订阅规则集缺少中国直连规则。

### 4. yay/AUR 报 proxyconnect 错误

`http_proxy` 指向 7897 但 Clash 没跑。用 `.zshrc` 里的别名：
```bash
ysync    # 系统升级直连
yrepo    # 安装官方包直连
yaur     # 安装 AUR 包走代理
```

### 5. 某个应用不读 http_proxy

部分应用（浏览器、Java 等）不读环境变量。依赖：
- 火狐：gsettings 系统代理
- 其他：需 TUN 模式（当前关闭中）

### 6. TUN 模式下国内网站超时

Clash gvisor TUN 在 Linux 上存在 DIRECT 路由死循环 bug。暂时关闭 TUN，用系统代理替代。
如需开启 TUN，需要 root 权限运行：
```bash
sudo ip rule add fwmark 0x163 lookup main pref 8999
```

## 有用的 Shell 命令

```bash
# 查看当前命中的规则
grep "using\|match" ~/.local/share/io.github.clash-verge-rev.clash-verge-rev/logs/service/service_latest.log | tail -10

# 查看所有 PROCESS-NAME 规则
grep "PROCESS-NAME" ~/.local/share/io.github.clash-verge-rev.clash-verge-rev/clash-verge.yaml

# 测试代理可达性
curl --proxy socks5h://127.0.0.1:7897 -s -o /dev/null -w "%{http_code}" https://目标地址

# 测试直连可达性
curl --noproxy '*' -s -o /dev/null -w "%{http_code}" https://目标地址

# 查看 Clash 实时连接
curl -s --unix-socket /tmp/verge/verge-mihomo.sock http://localhost/connections
```
