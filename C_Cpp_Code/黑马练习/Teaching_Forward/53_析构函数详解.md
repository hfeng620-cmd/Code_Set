# 析构函数详解

## 一、析构函数的作用

析构函数在**对象销毁前自动调用**，用于**清理资源**（释放堆区内存、关闭文件等）。

```cpp
#include <iostream>
using namespace std;

class Person {
public:
    string name;

    // 构造函数
    Person(string n) {
        name = n;
        cout << "[构造] " << name << " 被创建" << endl;
    }

    // 析构函数
    ~Person() {
        cout << "[析构] " << name << " 被销毁" << endl;
    }
};

int main() {
    cout << "--- main 开始 ---" << endl;

    Person p1("张三");
    Person p2("李四");

    cout << "--- main 即将结束 ---" << endl;
    return 0;
}
```

**输出：**
```
--- main 开始 ---
[构造] 张三 被创建
[构造] 李四 被创建
--- main 即将结束 ---
[析构] 李四 被销毁    ← 后构造的先析构
[析构] 张三 被销毁    ← 先构造的后析构
```

---

## 二、析构函数的特点

1. **函数名是 `~类名`**
2. **没有返回值**（和构造函数一样）
3. **没有参数**（不能重载，一个类只能有一个析构函数）
4. **对象销毁时自动调用**

```cpp
class Person {
public:
    string name;

    // 构造函数
    Person(string n) {
        name = n;
    }

    // 析构函数：~类名，没有返回值，没有参数
    ~Person() {
        cout << name << " 的析构函数" << endl;
    }

    // 错误示范：
    // ~Person(int a) {}   // ❌ 析构函数不能有参数
    // void ~Person() {}   // ❌ 析构函数不能有返回值
};
```

## 三、析构函数的调用时机

### 3.1 局部对象离开作用域时

```cpp
#include <iostream>
using namespace std;

class MyObj {
public:
    string name;

    MyObj(string n) : name(n) {
        cout << "[构造] " << name << endl;
    }

    ~MyObj() {
        cout << "[析构] " << name << endl;
    }
};

void test() {
    cout << "--- 进入 test 函数 ---" << endl;
    MyObj obj("test中的对象");
    cout << "--- 即将离开 test 函数 ---" << endl;
}

int main() {
    cout << "--- main 开始 ---" << endl;
    test();     // 调用 test 函数
    cout << "--- main 结束 ---" << endl;
    return 0;
}
```

**输出：**
```
--- main 开始 ---
--- 进入 test 函数 ---
[构造] test中的对象
--- 即将离开 test 函数 ---
[析构] test中的对象    ← 离开 test 函数时析构
--- main 结束 ---
```

### 3.2 delete 指向对象的指针时

```cpp
#include <iostream>
using namespace std;

class MyObj {
public:
    string name;

    MyObj(string n) : name(n) {
        cout << "[构造] " << name << endl;
    }

    ~MyObj() {
        cout << "[析构] " << name << endl;
    }
};

int main() {
    cout << "--- 在堆区创建对象 ---" << endl;
    MyObj *p = new MyObj("堆区对象");

    cout << "--- 准备 delete ---" << endl;
    delete p;       // 手动释放时调用析构函数
    cout << "--- delete 完成 ---" << endl;

    return 0;
}
```

**输出：**
```
--- 在堆区创建对象 ---
[构造] 堆区对象
--- 准备 delete ---
[析构] 堆区对象    ← delete 时调用析构
--- delete 完成 ---
```

### 3.3 程序结束时全局对象销毁

```cpp
#include <iostream>
using namespace std;

class MyObj {
public:
    string name;

    MyObj(string n) : name(n) {
        cout << "[构造] " << name << endl;
    }

    ~MyObj() {
        cout << "[析构] " << name << endl;
    }
};

// 全局对象，在 main 之前构造，在 main 之后析构
MyObj globalObj("全局对象");

int main() {
    cout << "--- main 开始 ---" << endl;
    MyObj localObj("局部对象");
    cout << "--- main 结束 ---" << endl;
    return 0;
}
```

**输出：**
```
[构造] 全局对象       ← main 之前就构造了
--- main 开始 ---
[构造] 局部对象
--- main 结束 ---
[析构] 局部对象       ← main 结束后，局部对象先析构
[析构] 全局对象       ← 最后全局对象析构
```

### 3.4 对象数组销毁时

```cpp
#include <iostream>
using namespace std;

class MyObj {
public:
    int id;

    MyObj(int i) : id(i) {
        cout << "[构造] 对象" << id << endl;
    }

    ~MyObj() {
        cout << "[析构] 对象" << id << endl;
    }
};

int main() {
    cout << "--- 创建对象数组 ---" << endl;
    MyObj arr[3] = {1, 2, 3};   // 创建3个对象

    cout << "--- 即将销毁 ---" << endl;
    return 0;
}
```

**输出：**
```
--- 创建对象数组 ---
[构造] 对象1
[构造] 对象2
[构造] 对象3
--- 即将销毁 ---
[析构] 对象3    ← 倒序析构
[析构] 对象2
[析构] 对象1
```

## 四、析构函数中应该做什么

析构函数主要用于**释放资源**：

1. **释放堆区内存**（如果有 `new` 的成员）
2. **关闭文件句柄**
3. **释放其他资源**（网络连接、锁等）

```cpp
#include <iostream>
#include <fstream>
using namespace std;

class DataManager {
private:
    int *m_data;         // 指向堆区的指针
    int m_size;

public:
    // 构造函数：在堆区分配内存
    DataManager(int size) {
        m_size = size;
        m_data = new int[m_size];    // 在堆区开辟数组
        cout << "构造：分配了 " << m_size << " 个int的堆区内存" << endl;
    }

    // 析构函数：释放堆区内存
    ~DataManager() {
        delete[] m_data;             // 释放堆区内存
        m_data = nullptr;            // 指针置空（好习惯）
        cout << "析构：释放了堆区内存" << endl;
    }

    // 设置数据
    void setData(int index, int value) {
        if (index >= 0 && index < m_size) {
            m_data[index] = value;
        }
    }

    // 获取数据
    int getData(int index) {
        if (index >= 0 && index < m_size) {
            return m_data[index];
        }
        return -1;
    }
};

int main() {
    DataManager dm(5);

    dm.setData(0, 100);
    dm.setData(1, 200);

    cout << "data[0] = " << dm.getData(0) << endl;
    cout << "data[1] = " << dm.getData(1) << endl;

    // dm 离开作用域时，析构函数自动调用，释放堆区内存
    return 0;
}
```

## 五、编译器默认提供的析构函数

如果你没有写析构函数，编译器会自动提供一个**空的析构函数**。

```cpp
class Person {
public:
    string name;
    int age;

    // 编译器自动提供：
    // ~Person() {}   // 空实现，什么都不做
};
```

**注意：** 如果类中有 `new` 分配的堆区内存，编译器默认的析构函数**不会释放**，必须自己写析构函数。

```cpp
class BadExample {
private:
    int *data;

public:
    BadExample() {
        data = new int[100];    // 堆区分配内存
    }

    // 如果不写析构函数：
    // ~BadExample() {}   // 编译器默认的空析构
    // data 指向的内存永远不会被释放 → 内存泄漏！
};

// 正确写法：
class GoodExample {
private:
    int *data;

public:
    GoodExample() {
        data = new int[100];
    }

    ~GoodExample() {
        delete[] data;      // 手动释放
        data = nullptr;
    }
};
```

## 六、构造和析构的完整顺序

**规则：先构造后析构，先子类后父类（涉及继承时）**

对于同层级的对象：**先构造的后析构，后构造的先析构**（栈的特性：先进后出）

```cpp
#include <iostream>
using namespace std;

class MyClass {
public:
    string name;

    MyClass(string n) : name(n) {
        cout << "[构造] " << name << endl;
    }

    ~MyClass() {
        cout << "[析构] " << name << endl;
    }
};

// 全局对象（最先构造，最后析构）
MyClass global1("全局对象A");

int main() {
    cout << "====== main 开始 ======" << endl;

    MyClass local1("局部对象1");

    {
        MyClass block1("代码块对象1");
        MyClass block2("代码块对象2");
        cout << "--- 代码块即将结束 ---" << endl;
    }   // block2 和 block1 在这里析构

    cout << "--- 代码块已结束 ---" << endl;

    MyClass local2("局部对象2");

    cout << "====== main 即将结束 ======" << endl;
    return 0;
}
```

**输出：**
```
[构造] 全局对象A       ← 最先构造（main之前）
====== main 开始 ======
[构造] 局部对象1
[构造] 代码块对象1
[构造] 代码块对象2
--- 代码块即将结束 ---
[析构] 代码块对象2     ← 后构造的先析构
[析构] 代码块对象1     ← 先构造的后析构
--- 代码块已结束 ---
[构造] 局部对象2
====== main 即将结束 ======
[析构] 局部对象2       ← 局部对象析构
[析构] 局部对象1
[析构] 全局对象A       ← 全局对象最后析构
```

## 七、构造和析构的代码验证

```cpp
#include <iostream>
using namespace std;

class Person {
public:
    string m_name;
    int m_age;

    // 无参构造
    Person() {
        m_name = "未知";
        m_age = 0;
        cout << "[无参构造] " << m_name << endl;
    }

    // 有参构造
    Person(string name, int age) {
        m_name = name;
        m_age = age;
        cout << "[有参构造] " << m_name << "，年龄" << m_age << endl;
    }

    // 拷贝构造
    Person(const Person &p) {
        m_name = p.m_name;
        m_age = p.m_age;
        cout << "[拷贝构造] 复制了 " << m_name << endl;
    }

    // 析构函数
    ~Person() {
        cout << "[析构] " << m_name << " 被销毁" << endl;
    }

    // 显示信息
    void show() {
        cout << "  " << m_name << "，" << m_age << "岁" << endl;
    }
};

int main() {
    cout << "===== 开始创建对象 =====" << endl;

    Person p1;                      // 无参构造
    Person p2("张三", 25);         // 有参构造
    Person p3(p2);                  // 拷贝构造

    cout << "\n===== 对象信息 =====" << endl;
    p1.show();
    p2.show();
    p3.show();

    cout << "\n===== 创建堆区对象 =====" << endl;
    Person *p4 = new Person("李四", 30);
    p4->show();

    cout << "\n===== 释放堆区对象 =====" << endl;
    delete p4;     // 手动释放，调用析构

    cout << "\n===== main 即将结束 =====" << endl;
    return 0;
}
```

**输出：**
```
===== 开始创建对象 =====
[无参构造] 未知
[有参构造] 张三，年龄25
[拷贝构造] 复制了 张三

===== 对象信息 =====
  未知，0岁
  张三，25岁
  张三，25岁

===== 创建堆区对象 =====
[有参构造] 李四，年龄30
  李四，30岁

===== 释放堆区对象 =====
[析构] 李四 被销毁    ← delete 触发析构

===== main 即将结束 =====
[析构] 张三 被销毁    ← 后构造的先析构
[析构] 张三 被销毁
[析构] 未知 被销毁    ← 先构造的后析构
```

## 八、堆区对象 vs 栈区对象的析构

```cpp
#include <iostream>
using namespace std;

class Demo {
public:
    string name;

    Demo(string n) : name(n) {
        cout << "[构造] " << name << endl;
    }

    ~Demo() {
        cout << "[析构] " << name << endl;
    }
};

int main() {
    cout << "=== 栈区对象 ===" << endl;
    Demo stackObj("栈区");    // 栈区对象，离开作用域自动析构

    cout << "\n=== 堆区对象 ===" << endl;
    Demo *heapObj = new Demo("堆区");  // 堆区对象，需要手动 delete

    cout << "\n=== 准备 delete ===" << endl;
    delete heapObj;     // 手动析构堆区对象

    cout << "\n=== main 结束 ===" << endl;
    return 0;
    // stackObj 在这里自动析构
}
```

**重要提醒：**
- **栈区对象**：离开作用域时自动析构，不需要手动管理
- **堆区对象**：必须手动 `delete` 才会析构，否则内存泄漏

## 九、常见错误和注意事项

### 错误1：忘记 delete 导致内存泄漏

```cpp
void test() {
    Person *p = new Person("张三");
    // 忘记 delete p;
    // 函数结束时，指针 p 被销毁，但堆区的 Person 对象还在
    // 没有任何指针指向它 → 内存泄漏！
}

// 正确写法：
void test() {
    Person *p = new Person("张三");
    // ... 使用 p ...
    delete p;       // 记得释放！
    p = nullptr;    // 置空（好习惯）
}
```

### 错误2：重复 delete

```cpp
Person *p = new Person("张三");
delete p;           // 第一次 delete，析构函数被调用
// delete p;        // ❌ 错误！重复 delete，程序崩溃！
p = nullptr;        // delete 后立即置空，可以避免这个问题
```

### 错误3：析构函数写了参数

```cpp
class Person {
public:
    // ~Person(int a) {}   // ❌ 错误！析构函数不能有参数
    ~Person() {}           // ✅ 正确
};
```

### 注意事项

1. 析构函数**不能有参数**，不能重载，一个类只有一个析构函数
2. 栈区对象**自动析构**，堆区对象需要**手动 delete**
3. 析构的顺序是**先构造后析构**（栈的特性）
4. 如果类中有 `new`，必须在析构中 `delete`
5. `delete` 后将指针**置为 nullptr** 是好习惯
