# 内存四区详解

## 一、为什么需要了解内存四区

C++ 程序运行时，操作系统会把内存分成**四个区域**来管理。
了解这四个区域，有助于理解变量的生命周期、作用域，以及避免常见的内存错误。

| 区域 | 存放内容 | 由谁管理 |
|------|---------|---------|
| 代码区 | 程序的机器指令 | 操作系统 |
| 全局区 | 全局变量、静态变量、常量 | 操作系统（程序结束释放） |
| 栈区 | 局部变量、函数参数、返回值 | 编译器自动管理 |
| 堆区 | 程序员手动分配的内存 | 程序员手动管理（new/delete） |

---

## 二、代码区

### 2.1 什么是代码区

程序编译后，所有代码（函数体、类的方法等）都会被转换成**机器指令**，
存放在代码区。

### 2.2 代码区的特点

```cpp
// 代码区的特点：
// 1. 共享：如果多次运行同一个程序，它们共享同一份代码区（节省内存）
// 2. 只读：防止程序意外修改自己的指令
```

```cpp
#include <iostream>
using namespace std;

void func1()
{
    cout << "func1 的代码存放在代码区" << endl;
}

void func2()
{
    cout << "func2 的代码也存放在代码区" << endl;
}

int main()
{
    // 函数的地址在代码区
    cout << "func1 的地址: " << (void *)func1 << endl;
    cout << "func2 的地址: " << (void *)func2 << endl;
    // 两个函数地址相邻，说明它们连续存放在代码区

    return 0;
}
```

## 三、全局区

### 3.1 什么是全局区

存放**全局变量**和**静态变量**，以及**字符串常量**和其他常量。
程序结束后由操作系统释放。

### 3.2 全局变量和局部变量的地址对比

```cpp
#include <iostream>
using namespace std;

// 全局变量：存放在全局区
int g_a = 10;
int g_b = 20;

// 全局常量
const int g_c = 30;
const int g_d = 40;

int main()
{
    // 局部变量：存放在栈区
    int a = 10;
    int b = 20;

    // 静态变量：存放在全局区
    static int s_a = 100;
    static int s_b = 200;

    // 局部常量
    const int c_a = 30;
    const int c_b = 40;

    // 字符串常量：存放在全局区
    const char *str = "hello";

    cout << "===== 全局区 =====" << endl;
    cout << "全局变量 g_a 地址: " << (long long)&g_a << endl;
    cout << "全局变量 g_b 地址: " << (long long)&g_b << endl;
    cout << "静态变量 s_a 地址: " << (long long)&s_a << endl;
    cout << "静态变量 s_b 地址: " << (long long)&s_b << endl;
    cout << "全局常量 g_c 地址: " << (long long)&g_c << endl;
    cout << "全局常量 g_d 地址: " << (long long)&g_d << endl;
    cout << "字符串常量 地址:   " << (long long)str << endl;

    cout << "\n===== 栈区 =====" << endl;
    cout << "局部变量 a 地址:   " << (long long)&a << endl;
    cout << "局部变量 b 地址:   " << (long long)&b << endl;
    cout << "局部常量 c_a 地址: " << (long long)&c_a << endl;
    cout << "局部常量 c_b 地址: " << (long long)&c_b << endl;

    // 运行后你会发现：
    // 全局区的地址互相靠近
    // 栈区的地址互相靠近
    // 两个区域的地址差距很大

    return 0;
}
```

### 3.3 全局区总结

```
全局区包含：
  ├── 全局变量（g_a, g_b）
  ├── 静态变量（static int s_a, s_b）
  ├── 字符串常量（"hello"）
  └── 全局常量（const int g_c, g_d）

不在全局区：
  ├── 局部变量（栈区）
  └── 局部常量（栈区）
```

## 四、栈区

### 4.1 什么是栈区

栈区由**编译器自动管理**，存放：
- 函数的**局部变量**
- 函数的**参数**
- 函数的**返回值**

函数执行结束，栈区的变量会**自动释放**。

### 4.2 栈区的生命周期

```cpp
#include <iostream>
using namespace std;

void func()
{
    int a = 10;   // a 在栈区，函数结束后自动销毁
    cout << "func 中 a = " << a << endl;
}

int main()
{
    func();
    // 函数调用结束后，func 中的 a 已经被销毁了

    // 下面的代码访问的是栈区中已经被释放的内存
    // 这是未定义行为，绝对不要这样做！
    return 0;
}
```

### 4.3 不要返回局部变量的地址

```cpp
#include <iostream>
using namespace std;

// 错误示范：返回局部变量的地址
int* func()
{
    int a = 10;     // 局部变量，函数结束后销毁
    return &a;      // 返回了已被销毁的地址！
}

int main()
{
    int *p = func();
    // p 指向的内存已经被释放了

    cout << *p << endl;  // 未定义行为：可能输出10，也可能输出垃圾值
    cout << *p << endl;  // 再读一次可能就变了

    // 为什么第一次可能"看起来正确"？
    // 因为那块内存还没有被其他数据覆盖，旧值还残留着
    // 但这只是巧合，不能依赖这种行为

    return 0;
}
```

### 4.4 栈区空间有限

```cpp
// 栈区大小一般只有 1~8 MB（取决于操作系统和编译器设置）
// 如果在栈上分配过大的数组，会导致栈溢出（Stack Overflow）

void func()
{
    // int arr[10000000];  // 大约 40MB，会导致栈溢出崩溃！
    // 大数组应该放在堆区
}
```

## 五、堆区

### 5.1 什么是堆区

堆区由**程序员手动管理**，使用 `new` 分配内存，使用 `delete` 释放内存。
堆区的空间很大，但需要自己负责释放，否则会造成**内存泄漏**。

### 5.2 在堆区创建数据

```cpp
#include <iostream>
using namespace std;

int* func()
{
    // 在堆区创建一个 int，值为 10
    int *p = new int(10);
    return p;   // 返回堆区地址，函数结束后不会被销毁
}

int main()
{
    int *p = func();
    cout << *p << endl;  // 10（堆区数据不会随函数结束而消失）

    // 使用完毕后必须手动释放
    delete p;
    // p 变成了悬空指针，建议置空
    p = nullptr;

    return 0;
}
```

### 5.3 堆区 vs 栈区

```cpp
#include <iostream>
using namespace std;

int main()
{
    // 栈区变量
    int a = 10;
    cout << "栈区变量地址: " << (long long)&a << endl;

    // 堆区变量
    int *p = new int(20);
    cout << "堆区变量地址: " << (long long)p << endl;

    // 堆区地址通常比栈区地址小（在 Windows 上）
    // 但这取决于操作系统和编译器

    delete p;
    return 0;
}
```

| 对比项 | 栈区 | 堆区 |
|--------|------|------|
| 管理者 | 编译器自动 | 程序员手动 |
| 空间大小 | 小（1~8 MB） | 大（取决于可用内存） |
| 分配速度 | 快 | 慢 |
| 释放方式 | 自动（函数结束） | 手动（delete） |
| 常见问题 | 栈溢出 | 内存泄漏 |
| 使用场景 | 小变量、临时数据 | 大数据、需要长期存在的数据 |

## 六、四区地址分布

```cpp
#include <iostream>
using namespace std;

// 全局变量
int g_var = 1;
const int g_const = 2;

int main()
{
    // 局部变量
    int l_var = 3;
    const int l_const = 4;

    // 静态变量
    static int s_var = 5;

    // 堆区
    int *heap = new int(6);

    // 字符串常量
    const char *str = "hello";

    cout << "===== 各区域地址 =====" << endl;
    cout << "代码区 (main函数):    " << (void *)main << endl;
    cout << "全局变量 g_var:       " << (long long)&g_var << endl;
    cout << "全局常量 g_const:     " << (long long)&g_const << endl;
    cout << "静态变量 s_var:       " << (long long)&s_var << endl;
    cout << "字符串常量:           " << (long long)str << endl;
    cout << "堆区数据:             " << (long long)heap << endl;
    cout << "局部变量 l_var:       " << (long long)&l_var << endl;
    cout << "局部常量 l_const:     " << (long long)&l_const << endl;

    // 一般情况下（Windows 64位）：
    // 代码区地址：最低
    // 全局区地址：较低（全局变量、静态变量、常量）
    // 堆区地址：   中等
    // 栈区地址：   最高

    delete heap;
    return 0;
}
```

### 典型地址分布图（从低到高）

```
低地址
  ┌──────────────┐
  │   代码区     │  程序的机器指令（只读、共享）
  ├──────────────┤
  │   全局区     │  全局变量、静态变量、字符串常量、全局常量
  ├──────────────┤
  │   堆区       │  new 分配的内存（向上增长）
  │     ↑↑↑      │
  │              │
  │     ↓↓↓      │
  │   栈区       │  局部变量、参数、返回值（向下增长）
  └──────────────┘
高地址
```

## 七、new 和堆区简介

```cpp
#include <iostream>
using namespace std;

int main()
{
    // 在堆区创建单个变量
    int *p1 = new int(100);
    cout << *p1 << endl;   // 100
    delete p1;

    // 在堆区创建数组
    int *p2 = new int[5]{10, 20, 30, 40, 50};
    for (int i = 0; i < 5; i++)
    {
        cout << p2[i] << " ";
    }
    cout << endl;
    delete[] p2;  // 释放数组用 delete[]

    return 0;
}
```

## 八、常见错误和注意事项

### 8.1 内存泄漏

```cpp
void memoryLeak()
{
    int *p = new int(10);
    // 忘记 delete p;
    // 函数结束后 p 被销毁，但堆区的内存没有释放
    // 那块内存就泄漏了，再也无法访问也无法释放
}

// 正确做法：
void noLeak()
{
    int *p = new int(10);
    // ... 使用 p ...
    delete p;     // 用完就释放
    p = nullptr;  // 置空防止悬空指针
}
```

### 8.2 悬空指针

```cpp
int *p = new int(10);
delete p;        // 释放了内存
// p 仍然保存着那块内存的地址，但它已经无效了
// *p = 20;      // 未定义行为！
p = nullptr;     // 释放后应该置空
```

### 8.3 重复释放

```cpp
int *p = new int(10);
delete p;
// delete p;     // 错误！重复释放会导致程序崩溃
p = nullptr;     // 置空后就不会误删了
```

## 总结

| 内存区域 | 存放内容 | 生命周期 | 管理者 |
|---------|---------|---------|--------|
| 代码区 | 机器指令 | 程序运行期间 | 操作系统 |
| 全局区 | 全局变量、静态变量、常量 | 程序运行期间 | 操作系统 |
| 栈区 | 局部变量、参数、返回值 | 函数调用期间 | 编译器 |
| 堆区 | new 分配的数据 | 手动控制 | 程序员 |

| 注意事项 | 说明 |
|---------|------|
| 不返回局部变量地址 | 栈区数据函数结束就销毁 |
| 不返回局部变量引用 | 同上，引用本质也是地址 |
| new 必须 delete | 否则内存泄漏 |
| delete 后置空 | 防止悬空指针 |
| 栈区不要开太大数组 | 会栈溢出，大数组用堆区 |
