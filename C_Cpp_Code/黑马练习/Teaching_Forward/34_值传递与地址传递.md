# 34 - 值传递与地址传递详解

## 一、什么是函数参数传递？

当我们调用一个函数时，需要把**实际的数据**传给函数内部使用。
这个"传数据"的方式，就叫做**参数传递**。

C++ 中有三种主要的传递方式：

| 传递方式 | 核心思想 | 能否修改实参 |
|----------|----------|-------------|
| 值传递 | 把数据的**副本**传进去 | 不能 |
| 地址传递 | 把数据的**地址**传进去 | 能 |
| 引用传递 | 给实参起一个**别名** | 能 |

---

## 二、值传递（Pass by Value）

### 2.1 什么是值传递？

值传递就是：**把实参的值拷贝一份，赋给形参**。

形参是实参的一个"复印件"，函数内部对形参的任何修改，都**不会影响**原来的实参。

### 2.2 代码示例

```cpp
#include <iostream>
using namespace std;

// 值传递：形参 a 和 b 是实参的副本
void mySwap(int a, int b)
{
    int temp = a;
    a = b;
    b = temp;
    // 这里 a 和 b 确实交换了
    // 但交换的是"副本"，跟 main 里的 x、y 没关系
    cout << "函数内 a = " << a << ", b = " << b << endl;
}

int main()
{
    int x = 10;
    int y = 20;

    cout << "调用前 x = " << x << ", y = " << y << endl;

    mySwap(x, y);  // x 和 y 的值被拷贝给 a 和 b

    cout << "调用后 x = " << x << ", y = " << y << endl;
    // x 还是 10，y 还是 20，没有交换！

    return 0;
}
```

**输出结果：**
```
调用前 x = 10, y = 20
函数内 a = 20, b = 10
调用后 x = 10, y = 20
```

### 2.3 图解内存关系

```
调用 mySwap(x, y) 时的内存情况：

  main 函数的栈帧              mySwap 函数的栈帧
  +---------+                 +---------+
  | x = 10  | ---拷贝--->    | a = 10  |
  +---------+                 +---------+
  | y = 20  | ---拷贝--->    | b = 20  |
  +---------+                 +---------+

  x 和 a 是两个独立的变量，各自占用不同的内存空间
  修改 a 不会影响 x
```

### 2.4 值传递的特点总结

- 实参和形参是**两个独立的变量**
- 函数内部对形参的修改**不影响实参**
- 优点：安全，不会意外修改外部数据
- 缺点：如果数据很大（如结构体、类对象），拷贝开销大

## 三、地址传递（Pass by Address）

### 3.1 什么是地址传递？

地址传递就是：**把实参的内存地址传给形参（指针）**。

形参是一个指针，它指向实参所在的内存地址。通过这个指针，函数内部**可以直接访问和修改**实参的数据。

### 3.2 代码示例

```cpp
#include <iostream>
using namespace std;

// 地址传递：形参是指针，接收实参的地址
void mySwap(int *p1, int *p2)
{
    int temp = *p1;   // 取出 p1 指向的值
    *p1 = *p2;        // 把 p2 指向的值赋给 p1 指向的地址
    *p2 = temp;       // 把 temp 赋给 p2 指向的地址
    // 这里通过指针直接修改了 main 里 x 和 y 的值
}

int main()
{
    int x = 10;
    int y = 20;

    cout << "调用前 x = " << x << ", y = " << y << endl;

    mySwap(&x, &y);  // 传入 x 和 y 的地址

    cout << "调用后 x = " << x << ", y = " << y << endl;
    // x 变成 20，y 变成 10，交换成功！

    return 0;
}
```

**输出结果：**
```
调用前 x = 10, y = 20
调用后 x = 20, y = 10
```

### 3.3 图解内存关系

```
调用 mySwap(&x, &y) 时的内存情况：

  main 函数的栈帧              mySwap 函数的栈帧
  +---------+                 +-----------+
  | x = 10  | <---*p1------- | p1 = &x   |  (p1 存的是 x 的地址)
  +---------+                 +-----------+
  | y = 20  | <---*p2------- | p2 = &y   |  (p2 存的是 y 的地址)
  +---------+                 +-----------+

  p1 和 p2 通过地址直接访问 x 和 y
  修改 *p1 就是修改 x，修改 *p2 就是修改 y
```

### 3.4 更多地址传递示例

```cpp
#include <iostream>
using namespace std;

// 通过地址传递修改数组元素
void changeArray(int *arr, int size)
{
    for (int i = 0; i < size; i++)
    {
        arr[i] = arr[i] * 2;  // 每个元素翻倍
        // arr[i] 等价于 *(arr + i)
    }
}

// 通过地址传递交换两个变量（封装成函数）
void swapValues(int *a, int *b)
{
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main()
{
    // 示例1：修改数组
    int nums[] = {1, 2, 3, 4, 5};
    changeArray(nums, 5);  // 数组名就是首地址
    cout << "修改后的数组：";
    for (int i = 0; i < 5; i++)
    {
        cout << nums[i] << " ";  // 输出：2 4 6 8 10
    }
    cout << endl;

    // 示例2：交换变量
    int a = 100, b = 200;
    swapValues(&a, &b);
    cout << "a = " << a << ", b = " << b << endl;  // a=200, b=100

    return 0;
}
```

### 3.5 地址传递的特点总结

- 形参是**指针**，存的是实参的地址
- 函数内部通过**解引用（*）**操作，可以直接修改实参
- 优点：可以修改实参；传地址比传大数据更高效
- 缺点：使用不当可能造成野指针、空指针等问题

## 四、引用传递（Pass by Reference）

### 4.1 什么是引用传递？

引用传递就是：**给实参起一个别名**，形参和实参指向同一块内存。

对形参的操作就是对实参的操作。

### 4.2 代码示例

```cpp
#include <iostream>
using namespace std;

// 引用传递：形参是实参的别名
void mySwap(int &a, int &b)
{
    int temp = a;
    a = b;
    b = temp;
    // a 就是 x 的别名，b 就是 y 的别名
    // 修改 a 就是修改 x
}

int main()
{
    int x = 10;
    int y = 20;

    cout << "调用前 x = " << x << ", y = " << y << endl;

    mySwap(x, y);  // 直接传变量名，不需要取地址

    cout << "调用后 x = " << x << ", y = " << y << endl;
    // x = 20, y = 10，交换成功！

    return 0;
}
```

### 4.3 引用传递的特点

- 形参是实参的**别名**，两者共用同一块内存
- 语法比指针更简洁（不需要 `&` 取地址和 `*` 解引用）
- 本质上是编译器帮你处理了指针的底层操作

## 五、大对象传值的开销问题

### 5.1 为什么值传递对大对象不好？

```cpp
#include <iostream>
#include <string>
using namespace std;

// 假设有一个很大的结构体
struct BigData
{
    int arr[1000];   // 4000 字节
    double d[500];   // 4000 字节
    string name;     // 约 32 字节
    // 总计约 8000+ 字节
};

// 值传递：每次调用都要拷贝 8000+ 字节！效率很低
void processByValue(BigData data)
{
    cout << "值传递处理: " << data.name << endl;
}

// 地址传递：只传一个指针（4或8字节），效率高
void processByAddress(BigData *data)
{
    cout << "地址传递处理: " << data->name << endl;
}

// 引用传递：不拷贝，效率和地址传递一样高
void processByReference(BigData &data)
{
    cout << "引用传递处理: " << data.name << endl;
}

int main()
{
    BigData myData;
    myData.name = "测试数据";

    processByValue(myData);        // 慢：拷贝整个结构体
    processByAddress(&myData);     // 快：只传地址
    processByReference(myData);    // 快：传引用

    return 0;
}
```

### 5.2 性能对比

```
假设结构体大小为 8000 字节：

值传递：   每次调用拷贝 8000 字节 → 慢，浪费内存
地址传递： 每次调用传 8 字节（64位指针）→ 快
引用传递： 每次调用传 8 字节（底层也是指针）→ 快

结论：
- 小数据（int, double等）：值传递就行
- 大数据（结构体、类对象）：用地址传递或引用传递
- 需要修改实参：用地址传递或引用传递
- 不需要修改实参但数据大：用 const 引用传递
```

### 5.3 const 引用传递（推荐做法）

```cpp
// 如果只需要读取，不需要修改，加 const 保护
void printData(const BigData &data)
{
    cout << data.name << endl;
    // data.name = "新值";  // 编译错误！const 保护，不能修改
}
```

## 六、三种传递方式对比表格

| 特性 | 值传递 | 地址传递 | 引用传递 |
|------|--------|---------|---------|
| 形参类型 | 普通变量 | 指针 `int *p` | 引用 `int &r` |
| 调用方式 | `func(a)` | `func(&a)` | `func(a)` |
| 函数内修改 | 不影响实参 | **影响实参** | **影响实参** |
| 语法复杂度 | 最简单 | 较复杂 | 较简单 |
| 大数据效率 | 低（要拷贝） | 高（只传地址） | 高（只传地址） |
| 是否可以为空 | 不适用 | 可以是 `nullptr` | 不能为 null |
| 是否可以重新指向 | 不适用 | 可以 | 不可以 |
| 底层实现 | 拷贝值 | 传地址 | 编译器内部用指针实现 |

---

## 七、选择建议

```
问自己三个问题：

1. 需要修改实参吗？
   - 不需要 → 值传递 或 const 引用传递
   - 需要  → 地址传递 或 引用传递

2. 数据大吗？（结构体、类对象、大数组）
   - 小数据 → 值传递
   - 大数据 → 引用传递（或 const 引用）

3. 参数可能为空吗？
   - 可能 → 地址传递（指针可以为 nullptr）
   - 不会 → 引用传递
```

## 八、综合练习

```cpp
#include <iostream>
using namespace std;

// 练习1：值传递 - 计算平方（不影响原值）
void squareByValue(int num)
{
    num = num * num;
    cout << "函数内: num = " << num << endl;
}

// 练习2：地址传递 - 通过指针修改原值
void squareByAddress(int *num)
{
    *num = (*num) * (*num);
}

// 练习3：引用传递 - 通过引用修改原值
void squareByReference(int &num)
{
    num = num * num;
}

// 练习4：const 引用 - 只读，不能修改
void printInfo(const string &name, int age)
{
    cout << "姓名: " << name << ", 年龄: " << age << endl;
    // name = "新名字";  // 编译错误！
}

int main()
{
    int a = 5;

    // 值传递
    squareByValue(a);
    cout << "main中 a = " << a << endl;  // a 仍然是 5

    // 地址传递
    squareByAddress(&a);
    cout << "地址传递后 a = " << a << endl;  // a 变成 25

    // 引用传递
    squareByReference(a);
    cout << "引用传递后 a = " << a << endl;  // a 变成 625

    // const 引用
    string name = "张三";
    printInfo(name, 18);

    return 0;
}
```

**输出结果：**
```
函数内: num = 25
main中 a = 5
地址传递后 a = 25
引用传递后 a = 625
姓名: 张三, 年龄: 18
```

## 九、常见错误

### 错误1：值传递想修改实参

```cpp
// 错误写法：想通过值传递交换两个变量
void mySwap(int a, int b)  // 值传递
{
    int temp = a;
    a = b;
    b = temp;
    // 这里只是交换了副本，main 里的变量没变！
}

// 正确写法：使用地址传递或引用传递
void mySwapCorrect(int &a, int &b)
{
    int temp = a;
    a = b;
    b = temp;
}
```

### 错误2：指针传参忘记取地址

```cpp
void func(int *p)
{
    cout << *p << endl;
}

int main()
{
    int a = 10;
    // func(a);   // 错误！a 不是地址
    func(&a);     // 正确！传入 a 的地址
    return 0;
}
```

### 错误3：返回局部变量的地址

```cpp
// 危险！返回局部变量的地址
int* dangerousFunc()
{
    int local = 42;
    return &local;  // local 在函数结束后被销毁，返回的地址无效！
}

// 正确做法：返回值，或者用动态内存
int safeFunc()
{
    int local = 42;
    return local;  // 返回值是安全的（拷贝了一份）
}
```

## 十、总结

- **值传递**：安全但不能改实参，大数据效率低
- **地址传递**：能改实参，需要掌握指针语法
- **引用传递**：能改实参，语法简洁，推荐使用
- 选择原则：**能用值传递就用值传递，需要修改就用引用传递，大数据用 const 引用传递**
