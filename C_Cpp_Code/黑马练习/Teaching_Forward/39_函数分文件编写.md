# 函数分文件编写

## 一、为什么需要分文件编写

### 1.1 单文件的问题

当程序变大时，所有代码写在一个文件里会带来很多问题：

```
单文件的问题：
1. 文件太长，难以阅读和维护
2. 多人协作困难（大家都改同一个文件，容易冲突）
3. 修改一个函数需要重新编译整个文件
4. 代码无法复用（其他项目想用某个函数，要复制粘贴）
5. 不符合软件工程的模块化思想
```

### 1.2 分文件的好处

```
分文件的好处：
1. 结构清晰：每个文件职责明确
2. 易于维护：修改某个功能只需要改对应的文件
3. 便于复用：其他项目 include 头文件即可使用
4. 支持协作：不同人负责不同文件
5. 编译效率：只重新编译修改过的文件
```

## 二、分文件编写的步骤

### 2.1 总体步骤

```
分文件编写需要创建三类文件：

1. 头文件（.h）    - 放函数声明、宏定义、类型定义等
2. 源文件（.cpp）  - 放函数的具体实现
3. 主文件（main.cpp）- 放主函数 main，通过 #include 引入头文件

文件关系图：
+----------------+     +----------------+     +----------------+
|   main.cpp     | --> |   myheader.h   | <-- |   mycode.cpp   |
|   (主函数)      |     |   (函数声明)    |     |   (函数实现)    |
|   #include .h  |     |                |     |   #include .h  |
+----------------+     +----------------+     +----------------+

编译时：main.cpp 和 mycode.cpp 分别编译成 .o 文件，然后链接在一起
```

### 2.2 步骤详解

```
步骤1：创建头文件（.h）
  - 写函数声明（只有函数头，没有函数体）
  - 写必要的 #include
  - 加上头文件保护

步骤2：创建源文件（.cpp）
  - #include 自己的头文件
  - 写函数的具体实现（函数体）

步骤3：修改主文件（main.cpp）
  - #include 自己的头文件
  - 直接使用头文件中声明的函数
```

## 三、完整示例：加减乘除计算器

### 3.1 头文件 calculator.h

```cpp
// calculator.h
// 这是计算器的头文件，包含所有函数的声明

#ifndef CALCULATOR_H    // 头文件保护开始：如果没定义过CALCULATOR_H
#define CALCULATOR_H    // 就定义CALCULATOR_H

#include <iostream>
using namespace std;

// ---- 函数声明 ----
// 只写函数头，不写函数体
// 分号结尾，表示"这个函数存在，具体实现在别处"

// 加法函数
double add(double a, double b);

// 减法函数
double subtract(double a, double b);

// 乘法函数
double multiply(double a, double b);

// 除法函数（注意除零检查）
double divide(double a, double b);

#endif  // 头文件保护结束
```

### 3.2 源文件 calculator.cpp

```cpp
// calculator.cpp
// 这是计算器的实现文件，包含所有函数的具体实现

#include "calculator.h"    // 引入自己的头文件（用双引号，不是尖括号）

// ---- 函数实现 ----
// 这里写完整的函数体

// 加法函数实现
double add(double a, double b) {
    return a + b;
}

// 减法函数实现
double subtract(double a, double b) {
    return a - b;
}

// 乘法函数实现
double multiply(double a, double b) {
    return a * b;
}

// 除法函数实现
double divide(double a, double b) {
    if (b == 0) {
        cout << "错误：除数不能为0！" << endl;
        return 0;
    }
    return a / b;
}
```

### 3.3 主文件 main.cpp

```cpp
// main.cpp
// 主程序文件

#include "calculator.h"    // 引入头文件，获得函数声明

int main() {
    double a = 10, b = 3;

    cout << a << " + " << b << " = " << add(a, b) << endl;
    cout << a << " - " << b << " = " << subtract(a, b) << endl;
    cout << a << " * " << b << " = " << multiply(a, b) << endl;
    cout << a << " / " << b << " = " << divide(a, b) << endl;

    // 测试除零
    cout << a << " / " << 0 << " = " << divide(a, 0) << endl;

    return 0;
}
```

### 3.4 编译命令

```bash
# 方法1：一步编译（简单项目推荐）
g++ main.cpp calculator.cpp -o calculator

# 方法2：分步编译（大项目推荐，只重新编译修改的文件）
g++ -c main.cpp -o main.o           # 编译main.cpp
g++ -c calculator.cpp -o calculator.o  # 编译calculator.cpp
g++ main.o calculator.o -o calculator  # 链接

# 运行
./calculator
```

**运行结果：**
```
10 + 3 = 13
10 - 3 = 7
10 * 3 = 30
10 / 3 = 3.33333
错误：除数不能为0！
10 / 0 = 0
```

## 四、头文件保护：#pragma once vs #ifndef

### 4.1 为什么需要头文件保护

```
问题：如果一个头文件被多次 #include 会怎样？

例如：
// a.h 中 #include "common.h"
// b.h 中 #include "common.h"
// main.cpp 中同时 #include "a.h" 和 #include "b.h"
// 结果 common.h 被包含了两次，函数声明重复，编译报错！

解决：头文件保护（Header Guard）
```

### 4.2 方式一：#ifndef（传统方式，所有编译器都支持）

```cpp
// myheader.h

#ifndef MYHEADER_H     // 如果 MYHEADER_H 没有被定义过
#define MYHEADER_H     // 那么定义它

// ... 头文件内容 ...
// 函数声明、类型定义等

#endif  // 结束 #ifndef
```

**工作原理：**
```
第一次 #include "myheader.h"：
  1. 检查 MYHEADER_H 是否定义 -> 没有
  2. 定义 MYHEADER_H
  3. 执行头文件内容

第二次 #include "myheader.h"：
  1. 检查 MYHEADER_H 是否定义 -> 已经定义了
  2. 跳过整个头文件内容
  3. 不会重复声明
```

### 4.3 方式二：#pragma once（现代方式，推荐）

```cpp
// myheader.h

#pragma once    // 告诉编译器：这个文件只包含一次

// ... 头文件内容 ...
// 函数声明、类型定义等
```

### 4.4 两种方式的对比

```
+--------------------+---------------------------+---------------------------+
| 特性               | #ifndef                   | #pragma once              |
+--------------------+---------------------------+---------------------------+
| 标准               | C/C++标准                 | 编译器扩展（非标准）       |
| 兼容性             | 所有编译器                 | 绝大多数现代编译器         |
| 写法               | 需要定义宏名，较繁琐       | 一行搞定                  |
| 性能               | 需要解析宏，稍慢           | 编译器直接处理，稍快       |
| 文件名冲突风险     | 宏名可能冲突               | 基本无冲突                |
| 推荐               | 需要极致兼容性时           | 日常开发推荐              |
+--------------------+---------------------------+---------------------------+

建议：在日常开发中使用 #pragma once，简洁方便。
```

## 五、头文件里写什么、不写什么

### 5.1 应该写在头文件里的内容

```cpp
// myheader.h
#pragma once

#include <iostream>
#include <string>
using namespace std;

// 1. #include 指令（需要的库）
// 2. 函数声明（只有函数头+分号）
void doSomething(int a, double b);
int calculate(int x);

// 3. 类的声明（后面会学到）
// class MyClass { ... };

// 4. 宏定义
#define MAX_SIZE 100

// 5. 类型别名
typedef unsigned long ulong;

// 6. 常量定义
const double PI = 3.14159;

// 7. 结构体定义
struct Point {
    int x;
    int y;
};
```

### 5.2 不应该写在头文件里的内容

```cpp
// myheader.h
#pragma once

// ====== 不要写这些！======

// 1. 不要写函数的实现（函数体）
// void doSomething(int a) {
//     cout << a << endl;    // 错误！实现应该放在 .cpp 文件中
// }

// 2. 不要定义全局变量（会导致重复定义错误）
// int globalVar = 100;      // 错误！应该用 extern int globalVar;

// 3. 不要写 using namespace std;（在头文件中不推荐，会污染包含者的命名空间）
// 不过对于初学者的小项目，可以接受

// 4. 不要写 main 函数
// int main() { return 0; }  // 错误！main 只能在主文件中
```

### 5.3 原则总结

```
头文件 (.h) 的原则：
  "声明"放在头文件，"实现"放在源文件

头文件放：
  - 函数声明（函数头 + 分号）
  - 类/结构体的定义
  - 宏定义、常量、类型别名
  - #include 需要的库

源文件 (.cpp) 放：
  - 函数的实现（完整的函数体）
  - #include 自己的头文件
```

## 六、多文件编译过程

### 6.1 编译流程图

```
源文件(.cpp)          预处理器           编译器              汇编器           链接器           可执行文件
                    (展开#include,     (语法检查,          (生成机器码)      (合并所有.o)
                     处理宏)           生成汇编代码)

main.cpp      ->   预处理main   ->   编译main    ->   main.o    -|
                                                              |->   链接   ->   program
calculator.cpp -> 预处理calc   ->   编译calc    ->   calc.o    -|
```

### 6.2 分步编译命令

```bash
# 步骤1：预处理（-E 选项，查看预处理结果）
g++ -E main.cpp -o main.i

# 步骤2：编译为汇编代码（-S 选项）
g++ -S main.cpp -o main.s

# 步骤3：编译为目标文件（-c 选项，最常用）
g++ -c main.cpp -o main.o
g++ -c calculator.cpp -o calculator.o

# 步骤4：链接为目标程序
g++ main.o calculator.o -o calculator

# 一步到位（省略中间步骤）
g++ main.cpp calculator.cpp -o calculator
```

## 七、常见错误及解决方案

### 7.1 错误1：头文件未加保护

```
错误信息：redefinition of 'xxx'（重定义）

原因：头文件被多次包含，导致函数声明重复

解决：加上 #pragma once 或 #ifndef 保护
```

```cpp
// 错误写法：
// myheader.h
void func();    // 如果被包含两次，就有两个相同的声明 -> 编译错误

// 正确写法：
// myheader.h
#pragma once
void func();    // 现在只会被处理一次
```

### 7.2 错误2：忘记包含头文件

```
错误信息：'xxx' was not declared in this scope

原因：使用了某个函数但没有 #include 它的声明

解决：在文件开头 #include 对应的头文件
```

### 7.3 错误3：函数声明和定义不一致

```
错误信息：undefined reference to 'xxx'

原因：
1. 函数声明和定义的签名不一致（参数类型、个数不同）
2. 忘记在项目中加入对应的 .cpp 文件
3. 函数只有声明没有定义

解决：检查函数签名是否完全一致，检查是否所有.cpp都参与编译
```

### 7.4 错误4：#include 路径问题

```cpp
// 使用尖括号 <>：在系统目录中查找
#include <iostream>     // 标准库头文件

// 使用双引号 ""：先在当前目录查找，再在系统目录查找
#include "myheader.h"   // 自己写的头文件
#include "calculator.h" // 自己写的头文件

// 如果头文件在子目录中：
#include "include/myheader.h"   // 指定相对路径
```

## 八、完整项目示例：学生管理系统

### 8.1 student.h（头文件）

```cpp
// student.h - 学生管理系统的头文件
#ifndef STUDENT_H
#define STUDENT_H

#include <iostream>
#include <string>
using namespace std;

// 学生结构体
struct Student {
    string name;
    int age;
    double score;
};

// 函数声明
void inputStudent(Student &s);           // 输入学生信息
void printStudent(const Student &s);     // 打印学生信息
double getScore(const Student &s);       // 获取成绩
bool isPass(const Student &s);           // 判断是否及格（60分以上）

#endif
```

### 8.2 student.cpp（实现文件）

```cpp
// student.cpp - 学生管理系统的实现
#include "student.h"

// 输入学生信息
void inputStudent(Student &s) {
    cout << "请输入姓名：";
    cin >> s.name;
    cout << "请输入年龄：";
    cin >> s.age;
    cout << "请输入成绩：";
    cin >> s.score;
}

// 打印学生信息
void printStudent(const Student &s) {
    cout << "姓名：" << s.name
         << "，年龄：" << s.age
         << "，成绩：" << s.score << endl;
}

// 获取成绩
double getScore(const Student &s) {
    return s.score;
}

// 判断是否及格
bool isPass(const Student &s) {
    return s.score >= 60.0;
}
```

### 8.3 main.cpp（主文件）

```cpp
// main.cpp - 主程序
#include "student.h"

int main() {
    Student stu;

    cout << "=== 输入学生信息 ===" << endl;
    inputStudent(stu);

    cout << "\n=== 学生信息 ===" << endl;
    printStudent(stu);

    cout << "\n=== 成绩判断 ===" << endl;
    if (isPass(stu)) {
        cout << stu.name << " 及格了！" << endl;
    } else {
        cout << stu.name << " 不及格，需要补考。" << endl;
    }

    return 0;
}
```

### 8.4 编译和运行

```bash
# 编译
g++ main.cpp student.cpp -o student_manager

# 运行
./student_manager
```

## 九、头文件包含的注意事项

### 9.1 include 的搜索路径

```cpp
// 1. 尖括号 <>：只在系统目录中搜索
#include <iostream>     // 搜索编译器的include目录

// 2. 双引号 ""：先搜索当前目录，再搜索系统目录
#include "myfile.h"     // 先找当前目录，找不到再找系统目录

// 所以：
// - 标准库用 <>
// - 自己的头文件用 ""
```

### 9.2 避免循环包含

```cpp
// 错误示例：循环包含
// a.h 中 #include "b.h"
// b.h 中 #include "a.h"
// 这会导致无限递归！使用 #ifndef 或 #pragma once 可以防止

// a.h
#pragma once
// #include "b.h"    // 如果 b.h 也包含了 a.h，就形成了循环

// 解决方案：
// 1. 重新设计头文件结构，消除循环依赖
// 2. 使用前向声明（forward declaration）
```

## 十、注意事项总结

| 序号 | 知识点 | 说明 |
|------|--------|------|
| 1 | 头文件放声明 | 函数头 + 分号，不写函数体 |
| 2 | 源文件放实现 | 完整的函数体 |
| 3 | 必须加头文件保护 | `#pragma once` 或 `#ifndef` |
| 4 | 自己的头文件用 `""` | `#include "myheader.h"` |
| 5 | 标准库用 `<>` | `#include <iostream>` |
| 6 | 声明和定义要一致 | 参数类型、个数、返回值必须相同 |
| 7 | 所有 .cpp 都要编译 | 编译时列出所有源文件 |
| 8 | 避免循环包含 | 重新设计或用前向声明 |

## 十一、练习题

1. 将一个简单的数学工具（求最大值、最小值、绝对值）分文件编写。
2. 创建一个字符串工具库，包含：字符串转大写、转小写、反转功能。
3. 思考：为什么 `#include` 标准库用尖括号 `<>`，自己的头文件用双引号 `""`？

```cpp
// 练习1 参考答案：

// ====== mathutils.h ======
#pragma once
#include <iostream>
using namespace std;

int myMax(int a, int b);
int myMin(int a, int b);
int myAbs(int x);

// ====== mathutils.cpp ======
#include "mathutils.h"

int myMax(int a, int b) {
    return (a > b) ? a : b;
}

int myMin(int a, int b) {
    return (a < b) ? a : b;
}

int myAbs(int x) {
    return (x < 0) ? -x : x;
}

// ====== main.cpp ======
#include "mathutils.h"

int main() {
    cout << "max(3, 7) = " << myMax(3, 7) << endl;
    cout << "min(3, 7) = " << myMin(3, 7) << endl;
    cout << "abs(-5) = " << myAbs(-5) << endl;
    return 0;
}

// 编译：g++ main.cpp mathutils.cpp -o mathapp
```
