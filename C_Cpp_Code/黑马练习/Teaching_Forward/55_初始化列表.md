# 初始化列表

## 一、初始化列表的语法

初始化列表是 C++ 提供的一种**在构造函数中初始化成员变量**的方式。

```cpp
构造函数(参数列表) : 成员1(值1), 成员2(值2), 成员3(值3) {
    // 构造函数体
}
```

### 基本示例

```cpp
#include <iostream>
using namespace std;

class Person {
public:
    string name;
    int age;
    double score;

    // 使用初始化列表
    Person(string n, int a, double s)
        : name(n), age(a), score(s)    // 初始化列表
    {
        cout << "构造函数被调用" << endl;
    }

    void show() {
        cout << "姓名：" << name << "，年龄：" << age << "，分数：" << score << endl;
    }
};

int main() {
    Person p("张三", 25, 95.5);
    p.show();

    return 0;
}
```

**输出：**
```
构造函数被调用
姓名：张三，年龄：25，分数：95.5
```

## 二、初始化列表 vs 构造函数体内赋值

### 2.1 构造函数体内赋值

```cpp
class Person {
public:
    string name;
    int age;

    Person(string n, int a) {
        // 先用默认值初始化（string 默认为空，int 默认为随机值）
        // 再赋值
        name = n;     // 赋值操作
        age = a;      // 赋值操作
    }
};
```

### 2.2 初始化列表

```cpp
class Person {
public:
    string name;
    int age;

    // 直接初始化，一步到位
    Person(string n, int a) : name(n), age(a) {
    }
};
```

### 2.3 效率对比

```cpp
class Example {
public:
    string str;

    // 方式1：构造函数体内赋值（效率低）
    // 步骤：1. 调用 string 默认构造（创建空字符串）
    //       2. 调用 string 的赋值运算符（赋新值）
    Example(string s) {
        str = s;     // 两步操作
    }

    // 方式2：初始化列表（效率高）
    // 步骤：1. 直接调用 string 的拷贝构造（一步到位）
    Example(string s) : str(s) {
        // 一步操作
    }
};
```

**总结：初始化列表效率更高，推荐使用。**

## 三、必须使用初始化列表的场景

### 3.1 const 成员变量

`const` 成员变量**必须在初始化列表中初始化**，不能在构造函数体内赋值。

```cpp
class Person {
public:
    const string m_id;    // const 成员，学号（不能修改）
    string m_name;

    // ✅ 正确：用初始化列表初始化 const 成员
    Person(string id, string name) : m_id(id), m_name(name) {
    }

    // ❌ 错误：不能在构造函数体内给 const 成员赋值
    // Person(string id, string name) {
    //     m_id = id;     // 编译错误！const 不能赋值
    //     m_name = name;
    // }
};

int main() {
    Person p("001", "张三");
    cout << "学号：" << p.m_id << endl;
    cout << "姓名：" << p.m_name << endl;

    return 0;
}
```

### 3.2 引用成员变量

引用成员变量**必须在初始化列表中初始化**。

```cpp
class Student {
public:
    string &m_name;     // 引用成员（绑定到外部变量）

    // ✅ 正确：用初始化列表初始化引用
    Student(string &name) : m_name(name) {
    }

    // ❌ 错误：不能在构造函数体内初始化引用
    // Student(string &name) {
    //     m_name = name;     // 编译错误！引用必须在声明时初始化
    // }
};

int main() {
    string name = "张三";
    Student s(name);

    cout << "姓名：" << s.m_name << endl;   // 输出：张三

    name = "李四";   // 修改原变量
    cout << "姓名：" << s.m_name << endl;   // 输出：李四（引用跟着变）

    return 0;
}
```

### 3.3 没有默认构造函数的类成员

如果类成员是另一个类的对象，且那个类**没有默认构造函数**，必须用初始化列表。

```cpp
class Engine {
public:
    int m_power;

    // 只有有参构造，没有无参构造
    Engine(int power) {
        m_power = power;
        cout << "Engine 构造：功率=" << m_power << endl;
    }
};

class Car {
public:
    string m_brand;
    Engine m_engine;    // 成员对象（Engine 没有默认构造函数）

    // ✅ 正确：用初始化列表调用 Engine 的有参构造
    Car(string brand, int power) : m_brand(brand), m_engine(power) {
    }

    // ❌ 错误：Engine 没有默认构造函数，编译器不知道怎么初始化 m_engine
    // Car(string brand, int power) {
    //     m_brand = brand;
    //     m_engine = power;   // 编译错误！
    // }
};

int main() {
    Car car("宝马", 300);
    cout << car.m_brand << "，功率" << car.m_engine.m_power << "匹" << endl;

    return 0;
}
```

## 四、初始化列表的执行顺序

**重要：初始化顺序按成员的声明顺序执行，不是初始化列表中的书写顺序！**

```cpp
#include <iostream>
using namespace std;

class Demo {
public:
    int m_a;
    int m_b;
    int m_c;

    // 注意：初始化列表中写的顺序是 c, b, a
    // 但实际执行顺序是按声明顺序：a, b, c
    Demo(int val)
        : m_c(val), m_b(m_c), m_a(m_b)    // 看起来应该从 c 开始
    {
        cout << "m_a = " << m_a << endl;
        cout << "m_b = " << m_b << endl;
        cout << "m_c = " << m_c << endl;
    }
};

int main() {
    Demo d(10);
    return 0;
}
```

**你可能期望输出：**
```
m_a = 10
m_b = 10
m_c = 10
```

**实际输出（可能是未定义行为）：**
```
m_a = 随机值    ← m_a 先初始化，但 m_b 还没初始化，是随机值
m_b = 随机值    ← m_b 第二个初始化，但 m_c 还没初始化，是随机值
m_c = 10        ← m_c 最后初始化，值为 10
```

**原因：** 成员按声明顺序 `m_a → m_b → m_c` 初始化，所以：
1. `m_a = m_b` → m_b 还没初始化，是随机值
2. `m_b = m_c` → m_c 还没初始化，是随机值
3. `m_c = 10` → 正确

**解决方法：** 让初始化列表的顺序和声明顺序一致，或者不要用前面的成员初始化后面的成员。

```cpp
class Correct {
public:
    int m_a;
    int m_b;
    int m_c;

    // 正确写法：初始化顺序和声明顺序一致
    Correct(int val)
        : m_a(val), m_b(m_a), m_c(m_b)    // 从前往后初始化
    {
        cout << "m_a = " << m_a << endl;   // 10
        cout << "m_b = " << m_b << endl;   // 10
        cout << "m_c = " << m_c << endl;   // 10
    }
};
```

## 五、更多代码示例

### 5.1 综合示例：Student 类

```cpp
#include <iostream>
#include <string>
using namespace std;

class Student {
private:
    const string m_id;      // const 成员（学号）
    string m_name;           // 姓名
    int m_age;               // 年龄
    double &m_scoreRef;      // 引用成员（绑定到外部分数）

public:
    // 初始化列表：初始化所有成员
    Student(string id, string name, int age, double &score)
        : m_id(id), m_name(name), m_age(age), m_scoreRef(score)
    {
        cout << "[构造] " << m_name << "（学号：" << m_id << "）" << endl;
    }

    void show() {
        cout << "  学号：" << m_id
             << "，姓名：" << m_name
             << "，年龄：" << m_age
             << "，分数：" << m_scoreRef << endl;
    }

    // 修改姓名
    void setName(string name) {
        m_name = name;
    }

    // 不能修改学号（const）
    // void setId(string id) { m_id = id; }   // ❌ 编译错误
};

int main() {
    double score = 95.0;
    Student s("2024001", "张三", 20, score);
    s.show();

    // 修改外部的 score
    score = 98.5;
    cout << "修改外部分数后：" << endl;
    s.show();    // s.m_scoreRef 也变了（因为是引用）

    return 0;
}
```

### 5.2 组合类的初始化列表

```cpp
#include <iostream>
using namespace std;

class Address {
public:
    string m_city;
    string m_street;

    // Address 只有有参构造
    Address(string city, string street) {
        m_city = city;
        m_street = street;
        cout << "[Address 构造] " << m_city << " " << m_street << endl;
    }
};

class Person {
public:
    string m_name;
    Address m_address;     // 组合：Person 包含 Address

    // 必须用初始化列表，因为 Address 没有默认构造函数
    Person(string name, string city, string street)
        : m_name(name), m_address(city, street)
    {
        cout << "[Person 构造] " << m_name << endl;
    }

    void show() {
        cout << "姓名：" << m_name
             << "，地址：" << m_address.m_city
             << " " << m_address.m_street << endl;
    }
};

int main() {
    Person p("张三", "北京", "长安街1号");
    p.show();

    return 0;
}
```

**输出：**
```
[Address 构造] 北京 长安街1号    ← 先初始化成员对象
[Person 构造] 张三                ← 再执行构造函数体
姓名：张三，地址：北京 长安街1号
```

## 六、初始化列表的简化写法

### C++11 支持在声明时直接初始化默认值

```cpp
class Person {
public:
    string m_name = "未命名";     // C++11：直接在声明时给默认值
    int m_age = 0;
    double m_score = 0.0;

    // 构造函数可以只初始化部分成员
    Person(string name) : m_name(name) {
        // m_age 和 m_score 使用声明时的默认值
    }

    Person() {
        // 所有成员使用声明时的默认值
    }
};

int main() {
    Person p1;              // 所有成员用默认值
    Person p2("张三");      // 只设置名字

    cout << p1.m_name;      // 输出：未命名
    cout << p2.m_name;      // 输出：张三
    cout << p2.m_age;       // 输出：0（默认值）

    return 0;
}
```

## 七、常见错误

### 错误1：const 成员在构造函数体内赋值

```cpp
class Bad {
public:
    const int m_val;

    Bad(int v) {
        // m_val = v;    // ❌ 错误！const 成员不能赋值
    }

    // 正确写法：
    // Bad(int v) : m_val(v) {}   // ✅ 初始化列表
};
```

### 错误2：引用成员在构造函数体内赋值

```cpp
class Bad {
public:
    int &m_ref;

    Bad(int &r) {
        // m_ref = r;    // ❌ 错误！引用必须在声明时绑定
    }

    // 正确写法：
    // Bad(int &r) : m_ref(r) {}   // ✅ 初始化列表
};
```

### 错误3：成员对象没有默认构造函数

```cpp
class Engine {
public:
    int m_power;
    Engine(int p) : m_power(p) {}   // 只有有参构造
};

class Car {
public:
    Engine m_engine;

    // Car(int p) {
    //     m_engine = Engine(p);   // ❌ 错误！Engine 没有默认构造
    // }

    // 正确写法：
    Car(int p) : m_engine(p) {}     // ✅ 初始化列表
};
```

### 错误4：初始化顺序依赖问题

```cpp
class Bad {
public:
    int m_a;
    int m_b;

    // m_a 先初始化（按声明顺序），此时 m_b 还没初始化
    Bad(int val) : m_b(val), m_a(m_b) {}   // ❌ m_a 的值是未定义的！
};
```

## 八、总结

| 对比项 | 初始化列表 | 构造函数体内赋值 |
|--------|-----------|-----------------|
| 效率 | 高（直接初始化） | 低（先默认初始化再赋值） |
| const 成员 | 可以初始化 | 不能赋值 |
| 引用成员 | 可以初始化 | 不能赋值 |
| 无默认构造的成员 | 可以初始化 | 不能赋值 |
| 推荐程度 | 推荐使用 | 特殊情况使用 |

**建议：养成使用初始化列表的习惯，既高效又能处理特殊情况。**
