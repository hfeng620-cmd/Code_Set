# C++ 浮点型（Floating-Point）详解

## 一、什么是浮点型？

浮点型就是用来存储**小数**（带小数点的数）的数据类型。
比如：`3.14, -0.5, 100.0, 2.5e3` 这些都是浮点数。

> "浮点"的意思是小数点可以"浮动"——可以在不同位置，用来表示很大或很小的数。

---

## 二、浮点型家族

C++ 提供了 3 种浮点型：

| 类型 | 关键字 | 占用字节 | 有效数字 | 大致范围 |
|------|--------|----------|----------|----------|
| 单精度 | `float` | 4 字节 | 6-7 位 | ±3.4×10³⁸ |
| 双精度 | `double` | 8 字节 | 15-16 位 | ±1.7×10³⁰⁸ |
| 长双精度 | `long double` | 12/16 字节 | 18-19 位 | 更大 |

```cpp
#include <iostream>
using namespace std;

int main() {
    // ============ 三种浮点型演示 ============

    // 1. float 单精度（4字节，6-7位有效数字）
    float f = 3.14f;  // f 后缀表示 float 类型
    cout << "float f = " << f << endl;
    cout << "float 占用字节数: " << sizeof(float) << endl;  // 4

    // 2. double 双精度（8字节，15-16位有效数字，最常用）
    double d = 3.141592653589793;
    cout << "double d = " << d << endl;
    cout << "double 占用字节数: " << sizeof(double) << endl;  // 8

    // 3. long double 长双精度
    long double ld = 3.141592653589793238L;  // L 后缀
    cout << "long double ld = " << ld << endl;
    cout << "long double 占用字节数: " << sizeof(long double) << endl;  // 12 或 16

    return 0;
}
```

## 三、float 与 double 的精度对比

### 3.1 什么是"有效数字"？

有效数字是指从左边第一个非零数字开始，到最后一位为止，一共多少位数字。

```cpp
#include <iostream>
#include <iomanip>  // 用于 setprecision
using namespace std;

int main() {
    // ============ 精度对比 ============

    // float 只能保证 6-7 位有效数字
    float f = 3.14159265f;
    cout << "float 值: " << f << endl;
    // 输出: 3.14159（后面的位数就不准确了）

    // double 能保证 15-16 位有效数字
    double d = 3.141592653589793;
    cout << "double 值: " << d << endl;
    // 输出: 3.141592653589793

    // 设置输出精度来观察区别
    cout << fixed << setprecision(20);  // 显示20位小数

    float f2 = 1.23456789f;
    double d2 = 1.23456789;

    cout << "float  1.23456789 -> " << f2 << endl;
    // 输出: 1.23456788063049316406（只有前7位准确）

    cout << "double 1.23456789 -> " << d2 << endl;
    // 输出: 1.23456789000000000000（完全准确）

    // 恢复默认精度
    cout << defaultfloat;

    return 0;
}
```

**输出：**
```
float  1.23456789 -> 1.23456788063049316406
double 1.23456789 -> 1.23456789000000000000
```

可以看到，float 的精度不够，第 7 位之后就不准确了。

### 3.2 精度丢失的直观演示

```cpp
#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    cout << fixed << setprecision(10);

    // float 精度不足导致的问题
    float f1 = 100000.12345f;
    float f2 = 100000.12346f;

    cout << "float f1 = " << f1 << endl;
    cout << "float f2 = " << f2 << endl;
    cout << "f1 == f2 ? " << (f1 == f2 ? "相等" : "不相等") << endl;
    // 可能输出"相等"！因为 float 精度不够，分不清这两个数

    // double 就没问题
    double d1 = 100000.12345;
    double d2 = 100000.12346;
    cout << "double d1 = " << d1 << endl;
    cout << "double d2 = " << d2 << endl;
    cout << "d1 == d2 ? " << (d1 == d2 ? "相等" : "不相等") << endl;
    // 输出"不相等"

    cout << defaultfloat;
    return 0;
}
```

## 四、浮点数的精度问题（重点！）

### 4.1 为什么 0.1 + 0.2 != 0.3？

这是所有编程语言都会遇到的问题，不是 C++ 的 bug。

```cpp
#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    // ============ 经典问题：0.1 + 0.2 = ? ============

    double a = 0.1;
    double b = 0.2;
    double c = a + b;

    cout << fixed << setprecision(20);
    cout << "0.1       = " << a << endl;
    cout << "0.2       = " << b << endl;
    cout << "0.1 + 0.2 = " << c << endl;
    cout << "0.3       = " << 0.3 << endl;

    cout << "0.1 + 0.2 == 0.3 ? " << (c == 0.3 ? "是" : "不是") << endl;
    // 输出：不是！

    cout << defaultfloat;
    return 0;
}
```

**输出：**
```
0.1       = 0.10000000000000000555
0.2       = 0.20000000000000001110
0.1 + 0.2 = 0.30000000000000004441
0.3       = 0.29999999999999998890
0.1 + 0.2 == 0.3 ? 不是
```

### 4.2 为什么会这样？—— IEEE 754 标准简介

计算机用**二进制**存储小数，但很多十进制小数无法精确表示为二进制小数。

```
类比：就像 1/3 = 0.333333... 无法用有限小数表示一样

十进制 0.1 转二进制：
0.1 × 2 = 0.2  → 取 0
0.2 × 2 = 0.4  → 取 0
0.4 × 2 = 0.8  → 取 0
0.8 × 2 = 1.6  → 取 1
0.6 × 2 = 1.2  → 取 1
0.2 × 2 = 0.4  → 取 0（开始循环！）
...
结果：0.0001100110011...（无限循环）

所以 0.1 在计算机中是近似值，不是精确值！
```

### 4.3 如何正确比较浮点数？

```cpp
#include <iostream>
#include <cmath>  // 包含 fabs 函数
using namespace std;

int main() {
    // ====== 错误做法：直接用 == 比较浮点数 ======
    double a = 0.1 + 0.2;
    double b = 0.3;

    if (a == b) {
        cout << "相等" << endl;  // 不会执行！
    } else {
        cout << "不相等（直接比较）" << endl;  // 会执行
    }

    // ====== 正确做法：判断差值是否足够小 ======
    double epsilon = 1e-9;  // 一个非常小的数（0.000000001）

    if (fabs(a - b) < epsilon) {  // fabs 是求绝对值
        cout << "近似相等（推荐做法）" << endl;  // 会执行
    } else {
        cout << "不相等" << endl;
    }

    // 解释：如果两个数的差值小于一个极小值，
    // 我们就认为它们"足够接近"，可以视为相等

    return 0;
}
```

## 五、科学计数法表示

C++ 支持用科学计数法（也叫指数表示法）来表示浮点数。

```cpp
#include <iostream>
using namespace std;

int main() {
    // ============ 科学计数法 ============

    // 格式：数字e指数 或 数字E指数
    // e/E 表示 "乘以10的几次方"

    // 正指数（大数）
    double a = 3e2;     // 3 × 10² = 300
    double b = 3E2;     // 同上，大写 E 也行
    double c = 1.5e6;   // 1.5 × 10⁶ = 1500000
    double d = 2.5e10;  // 2.5 × 10¹⁰ = 25000000000

    cout << "3e2 = " << a << endl;      // 300
    cout << "3E2 = " << b << endl;      // 300
    cout << "1.5e6 = " << c << endl;    // 1500000
    cout << "2.5e10 = " << d << endl;   // 25000000000

    // 负指数（小数）
    double e = 3e-2;    // 3 × 10⁻² = 0.03
    double f = 1.5e-3;  // 1.5 × 10⁻³ = 0.0015
    double g = 2.5e-6;  // 2.5 × 10⁻⁶ = 0.0000025

    cout << "3e-2 = " << e << endl;     // 0.03
    cout << "1.5e-3 = " << f << endl;   // 0.0015
    cout << "2.5e-6 = " << g << endl;   // 0.0000025

    // 科学计数法常用于表示非常大或非常小的数
    double light_speed = 3e8;        // 光速：3×10⁸ m/s
    double planck = 6.626e-34;       // 普朗克常数
    double earth_mass = 5.972e24;    // 地球质量（kg）

    cout << "光速: " << light_speed << " m/s" << endl;
    cout << "普朗克常数: " << planck << endl;
    cout << "地球质量: " << earth_mass << " kg" << endl;

    return 0;
}
```

**输出：**
```
3e2 = 300
3E2 = 300
1.5e6 = 1500000
2.5e10 = 25000000000
3e-2 = 0.03
1.5e-3 = 0.0015
2.5e-6 = 2.5e-06
光速: 3e+08 m/s
普朗克常数: 6.626e-34
地球质量: 5.972e+24 kg
```

## 六、取模运算 % 只能用于整型

取模运算（求余数）**不能**用于浮点型！

```cpp
#include <iostream>
using namespace std;

int main() {
    // ============ 取模运算的限制 ============

    // 正确：整型可以取模
    int a = 10;
    int b = 3;
    cout << "10 % 3 = " << a % b << endl;  // 输出 1（余数）

    // 错误：浮点型不能取模！
    // double x = 10.5;
    // double y = 3.0;
    // cout << x % y << endl;  // 编译错误！% 不能用于浮点数

    // 如果需要对浮点数取模，用 fmod 函数
    #include <cmath>
    double x = 10.5;
    double y = 3.0;
    double remainder = fmod(x, y);  // 用 fmod 函数
    cout << "fmod(10.5, 3.0) = " << remainder << endl;  // 1.5

    return 0;
}
```

## 七、sizeof 查看浮点型大小

```cpp
#include <iostream>
using namespace std;

int main() {
    // ============ 用 sizeof 查看浮点型大小 ============

    cout << "float 大小:       " << sizeof(float) << " 字节" << endl;       // 4
    cout << "double 大小:      " << sizeof(double) << " 字节" << endl;      // 8
    cout << "long double 大小: " << sizeof(long double) << " 字节" << endl; // 12 或 16

    // 也可以用变量
    float f = 3.14f;
    double d = 3.14;
    cout << "变量 f 大小: " << sizeof(f) << " 字节" << endl;  // 4
    cout << "变量 d 大小: " << sizeof(d) << " 字节" << endl;  // 8

    // 注意：sizeof 不需要括号也能用于变量
    cout << sizeof f << endl;  // 也可以（但不推荐，建议加括号）

    // 对比整型大小
    cout << "\n=== 整型和浮点型大小对比 ===" << endl;
    cout << "short:       " << sizeof(short) << " 字节" << endl;       // 2
    cout << "int:         " << sizeof(int) << " 字节" << endl;         // 4
    cout << "long long:   " << sizeof(long long) << " 字节" << endl;   // 8
    cout << "float:       " << sizeof(float) << " 字节" << endl;       // 4
    cout << "double:      " << sizeof(double) << " 字节" << endl;      // 8

    return 0;
}
```

## 八、浮点数的特殊值

```cpp
#include <iostream>
#include <cmath>
#include <limits>
using namespace std;

int main() {
    // ============ 浮点数的特殊值 ============

    // 1. 无穷大（Infinity）
    double inf = 1.0 / 0.0;  // 正无穷
    cout << "1.0 / 0.0 = " << inf << endl;       // inf
    cout << "-1.0 / 0.0 = " << -1.0 / 0.0 << endl; // -inf

    // 用 INFINITY 宏表示
    cout << "INFINITY = " << INFINITY << endl;

    // 2. 非数（NaN, Not a Number）
    double nan_val = sqrt(-1.0);  // 负数开平方
    cout << "sqrt(-1) = " << nan_val << endl;  // nan

    // 3. 判断是否是 NaN
    // 注意：NaN 不等于任何值，包括它自己！
    cout << "nan == nan ? " << (nan_val == nan_val ? "是" : "不是") << endl;
    // 输出"不是"！
    // 正确的判断方法：
    cout << "isnan(nan) ? " << (isnan(nan_val) ? "是" : "不是") << endl;

    // 4. 判断是否是无穷大
    cout << "isinf(inf) ? " << (isinf(inf) ? "是" : "不是") << endl;

    return 0;
}
```

## 九、float 和 double 的选择建议

```
1. 一般情况 → 用 double（精度高，现代计算机处理 double 和 float 速度差不多）
2. 需要节省内存（大量浮点数据）→ 用 float
3. 科学计算、金融计算 → 用 double（或更高精度）
4. 除非有特殊原因，不要用 float（精度不够容易出bug）
5. 避免用 == 直接比较浮点数，用差值比较法
```

## 十、常见错误和注意事项

### 错误1：忘记 f 后缀

```cpp
float f1 = 3.14;   // 警告！3.14 默认是 double，赋给 float 可能丢失精度
float f2 = 3.14f;  // 正确！f 后缀表示 float 类型
```

### 错误2：直接比较浮点数

```cpp
double a = 0.1 + 0.2;
double b = 0.3;

// 错误做法
if (a == b) { ... }  // 可能不成立！

// 正确做法
if (fabs(a - b) < 1e-9) { ... }  // 判断差值是否足够小
```

### 错误3：浮点数用 % 取模

```cpp
// 错误
// double a = 10.0 % 3.0;  // 编译错误！

// 正确
#include <cmath>
double result = fmod(10.0, 3.0);  // 用 fmod 函数
```

### 错误4：float 精度不足

```cpp
float f = 123456.789f;
cout << f << endl;  // 输出可能不准确，因为 float 只有 6-7 位有效数字

// 改用 double
double d = 123456.789;
cout << d << endl;  // 准确
```

## 十一、总结

| 项目 | 要点 |
|------|------|
| 最常用 | `double`（8字节，15-16位有效数字） |
| float | 4字节，6-7位有效数字，需要 f 后缀 |
| 精度问题 | 0.1 + 0.2 != 0.3，这是所有语言的通病 |
| 比较浮点数 | 不要用 ==，要用 fabs(a-b) < epsilon |
| 科学计数法 | 3e2 = 300，3e-2 = 0.03 |
| 取模 | % 只能用于整型，浮点用 fmod() |
| 选择建议 | 一般用 double，除非有特殊需求 |
