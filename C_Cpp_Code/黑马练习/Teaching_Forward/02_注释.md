# 02 注释详解

## 一、什么是注释

- 注释是写给 **人看的**，编译器会 **完全忽略** 注释内容
- 注释的目的是 **解释代码的意图**，让别人（包括未来的自己）更容易理解
- 好的注释能大大降低代码维护成本

## 二、单行注释 //

### 2.1 基本用法

```cpp
// 这是一行注释，编译器会忽略这行内容

int age = 18;  // 在代码后面也可以加注释，说明这行代码的作用

// 注释可以单独占一行，用来解释下面的代码
int score = 100;
```

### 2.2 常见使用场景

```cpp
// 场景1：解释变量含义
int totalScore = 0;    // 总分
int count = 0;          // 人数
double avgScore = 0.0;  // 平均分

// 场景2：解释代码逻辑
if (score >= 60)
{
    // 分数大于等于60表示及格
    cout << "及格" << endl;
}
else
{
    // 分数小于60表示不及格
    cout << "不及格" << endl;
}

// 场景3：标注待完成的事项
// TODO: 这里还需要添加输入验证功能
// FIXME: 这里的计算公式可能有精度问题
// HACK: 临时解决方案，后续需要优化
```

### 2.3 注释掉代码（调试常用）

```cpp
// 调试时，可以临时注释掉某行代码，让它不执行
int a = 10;
// cout << a << endl;   // 临时注释掉，不执行这行
int b = 20;
cout << b << endl;
```

## 三、多行注释 /* */

### 3.1 基本用法

```cpp
/* 这是一个多行注释
   可以跨越多行
   常用于写大段的说明文字 */

/*
    这也是多行注释
    通常每行前面加 * 号，更美观
    但这只是风格习惯，不加也完全可以
*/
```

### 3.2 常见使用场景

```cpp
/*
 * 函数名：calculateArea
 * 功能：计算矩形面积
 * 参数：width - 宽度, height - 高度
 * 返回值：矩形的面积
 * 作者：张三
 * 日期：2024-01-01
 */
double calculateArea(double width, double height)
{
    return width * height;
}
```

```cpp
/*
 * 注意事项：
 * 1. 传入的半径不能为负数
 * 2. 返回值是 double 类型，注意精度
 * 3. 使用 3.14159 近似值
 */
double calculateCircleArea(double radius)
{
    return 3.14159 * radius * radius;
}
```

## 四、注释不能嵌套（重要！）

### 4.1 错误示例

```cpp
/* 外层注释开始
   /* 内层注释 */   // 这里内层注释的 */ 会结束外层注释！
   这行代码就暴露在外面了！编译会报错！
   外层注释结束 */
```

> `/* */` 不能嵌套！编译器遇到第一个 `*/` 就认为注释结束了。

### 4.2 正确做法

```cpp
// 如果需要注释掉一大段已经包含 /* */ 注释的代码
// 请使用 // 逐行注释，或者把里面的 /* */ 先去掉

// 方法1：用 // 逐行注释
// /* 内层注释 */
// 这行代码
// /* 又一个内层注释 */

// 方法2：用 #if 0 预处理（编译器直接忽略整段）
#if 0
/* 内层注释 */
这段代码会被完全忽略
/* 另一个内层注释 */
#endif
```

## 五、注释的好习惯

### 5.1 应该注释什么

```cpp
// 好的注释：解释"为什么"这样做
// 不好的注释：解释"做了什么"（代码本身已经说明了做了什么）

// 不好的注释（废话注释）：
int age = 18;   // 把 18 赋值给 age（这还用你说？）

// 好的注释（解释意图）：
int age = 18;   // 默认年龄设为18，表示成年用户
```

### 5.2 函数注释模板

```cpp
/**
 * @brief  计算两个数的最大值
 * @param  a  第一个数
 * @param  b  第二个数
 * @return 两个数中较大的那个
 * @note   如果两个数相等，返回第一个参数
 */
int getMax(int a, int b)
{
    return (a > b) ? a : b;
}
```

### 5.3 类/结构体注释模板

```cpp
/**
 * @brief 学生信息结构体
 * @details 用于存储学生的基本信息，包括姓名、年龄、成绩
 */
struct Student
{
    string name;     // 学生姓名
    int age;         // 学生年龄
    double score;    // 考试成绩（满分100）
};
```

### 5.4 关键算法注释

```cpp
// 冒泡排序：相邻元素两两比较，每轮把最大的"冒泡"到末尾
void bubbleSort(int arr[], int n)
{
    for (int i = 0; i < n - 1; i++)         // 外层：控制轮数
    {
        for (int j = 0; j < n - 1 - i; j++) // 内层：每轮比较次数
        {
            // 如果前面的数比后面大，就交换
            if (arr[j] > arr[j + 1])
            {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}
```

## 六、注释的反面教材

### 6.1 过度注释（噪音注释）

```cpp
// 不要这样写！注释太多反而干扰阅读
int a = 10;    // 定义变量a
int b = 20;    // 定义变量b
int c = a + b; // c等于a加b
cout << c;     // 输出c

// 应该这样写：让代码自解释，必要时才加注释
int price = 10;       // 单价
int quantity = 20;    // 数量
int total = price + quantity;  // 计算总价
cout << total << endl;         // 输出总价
```

### 6.2 过时注释（误导性注释）

```cpp
// 错误：代码改了，注释没改！
int count = 0;   // 初始值设为1  <- 注释说1，实际是0，会误导人！

// 规则：修改代码时，一定要同步更新注释
int count = 0;   // 初始计数为0
```

## 七、实战：带完整注释的程序

```cpp
#include <iostream>
using namespace std;

/*
 * 程序名称：简易计算器
 * 功能：实现两个整数的加减乘除
 * 作者：学习笔记
 * 日期：2024
 */

int main()
{
    // 声明两个操作数
    int num1, num2;

    // 提示用户输入
    cout << "请输入两个整数（用空格分隔）: ";
    cin >> num1 >> num2;  // 从键盘读取两个整数

    // 计算并输出四种运算结果
    cout << "加法: " << num1 << " + " << num2 << " = " << num1 + num2 << endl;
    cout << "减法: " << num1 << " - " << num2 << " = " << num1 - num2 << endl;
    cout << "乘法: " << num1 << " * " << num2 << " = " << num1 * num2 << endl;

    // 除法需要判断除数是否为0
    if (num2 != 0)
    {
        // 除数不为0，正常计算
        // 注意：整数除法会丢弃小数部分
        cout << "除法: " << num1 << " / " << num2 << " = " << num1 / num2 << endl;
    }
    else
    {
        // 除数为0，提示错误
        cout << "除法: 错误！除数不能为0！" << endl;
    }

    return 0;  // 程序正常结束
}
```

## 八、总结

| 类型 | 语法 | 用途 | 能否嵌套 |
|------|------|------|----------|
| 单行注释 | `//` | 简短说明，解释单行代码 | 可以放在 `/* */` 内部 |
| 多行注释 | `/* */` | 大段说明，函数/类文档 | 不能嵌套 `/* */` |

> 记住：**好的代码应该是自解释的**，注释是补充而非替代。写注释的原则是：**解释为什么，而不是做了什么**。
