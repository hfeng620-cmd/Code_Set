# 04 常量详解

## 一、什么是常量

常量是 **不可修改的值**。一旦定义，程序运行期间不能改变。C++ 中定义常量有两种方式：

| 方式 | 关键字/指令 | 是否推荐 |
|------|-------------|----------|
| 宏常量 | `#define` | 不推荐（旧式 C 风格） |
| const 常量 | `const` | 推荐（类型安全） |

---

## 二、#define 宏常量

### 2.1 基本语法

```cpp
// 语法：#define 常量名 常量值
// 注意：没有分号，没有等号，没有类型

#define PI 3.14159
#define MAX_SIZE 100
#define SCHOOL_NAME "清华大学"
```

### 2.2 使用示例

```cpp
#include <iostream>
using namespace std;

#define PI 3.14159
#define MAX_SCORE 100

int main()
{
    double radius = 5.0;
    double area = PI * radius * radius;  // 编译时 PI 会被替换为 3.14159
    cout << "圆的面积: " << area << endl;

    int score = 95;
    if (score >= MAX_SCORE)
    {
        cout << "满分！" << endl;
    }

    return 0;
}
```

### 2.3 #define 的本质：文本替换

```cpp
#define PI 3.14159

// 编译器在预处理阶段，会把代码中所有 "PI" 直接替换为 "3.14159"
// 所以下面的代码：
double area = PI * r * r;
// 经过预处理后变成了：
double area = 3.14159 * r * r;

// 注意：这是纯粹的 文本替换，不涉及任何类型检查！
#define A 100
// 如果写成下面这样，预处理后会出问题：
#define B 10 + 20
int result = B * 2;   // 替换后变成: int result = 10 + 20 * 2;
// 想想结果是多少？不是 60，而是 50！因为先算乘法！

// 解决方法：加括号
#define B_CORRECT (10 + 20)
int result2 = B_CORRECT * 2;  // 替换后: int result2 = (10 + 20) * 2; = 60
```

### 2.4 #define 的坑

```cpp
// 坑1：没有类型检查
#define AGE 18
// AGE 可以被用在任何地方，编译器不会检查它是否合理
// 如果不小心写成: #define AGE "abc"，也不会报错，只有在使用时才出错

// 坑2：没有作用域限制
#define MAX 100  // 定义后，从这行开始到文件结束都有效
               // 不能用 {} 限制作用域

// 坑3：调试困难
// 调试时看不到宏的名字，只能看到替换后的值

// 坑4：可能污染其他代码
#define count 10  // 如果其他库也用了 count 这个名字，就会冲突
```

## 三、const 修饰的常量

### 3.1 基本语法

```cpp
// 语法：const 数据类型 常量名 = 常量值;
// 特点：有类型、有作用域、不能修改

const double PI = 3.14159;
const int MAX_SCORE = 100;
const string SCHOOL = "清华大学";
```

### 3.2 使用示例

```cpp
#include <iostream>
using namespace std;

int main()
{
    const int MAX_AGE = 150;
    const double TAX_RATE = 0.08;

    int age = 25;
    if (age <= MAX_AGE)
    {
        cout << "年龄合法" << endl;
    }

    double price = 100.0;
    double tax = price * TAX_RATE;
    cout << "税额: " << tax << endl;

    // 尝试修改 const 常量（编译报错！）
    // MAX_AGE = 200;  // 错误！不能修改 const 常量

    return 0;
}
```

### 3.3 const 的特点

```cpp
// 特点1：有类型检查
const int a = 10;    // a 是 int 类型
const double b = 3.14; // b 是 double 类型
// 编译器会检查类型是否匹配

// 特点2：有作用域
void func()
{
    const int x = 10;  // x 只在 func 函数内有效
}

int main()
{
    // cout << x;  // 错误！x 不在这个作用域
    return 0;
}

// 特点3：编译器可能会优化
// 对于简单的 const 值，编译器会在编译时直接替换，效率和宏一样
```

## 四、#define 和 const 的详细对比

| 对比项 | #define 宏常量 | const 常量 |
|--------|----------------|------------|
| 本质 | 文本替换（预处理阶段） | 有类型的变量（编译阶段） |
| 类型检查 | 无 | 有（编译器会检查） |
| 作用域 | 从定义处到文件末尾 | 遵循普通变量的作用域规则 |
| 调试 | 困难（看不到宏名） | 方便（有变量名和类型） |
| 内存 | 不分配内存（直接替换） | 可能分配内存（也可能被优化掉） |
| 推荐度 | 不推荐 | 推荐 |

### 代码对比

```cpp
// ====== #define 的写法 ======
#define PI 3.14159          // 没有类型，没有分号
#define MAX 100             // 没有类型，没有分号

// ====== const 的写法 ======
const double PI = 3.14159;  // 有类型 double
const int MAX = 100;         // 有类型 int

// const 更安全，因为编译器会帮你检查：
const int age = "hello";    // 编译报错！类型不匹配
#define AGE "hello"          // 不报错，只有使用时才可能出错
```

## 五、使用场景

### 5.1 适合用 #define 的场景

```cpp
// 场景1：条件编译（这是 #define 独有的功能）
#define DEBUG_MODE

#ifdef DEBUG_MODE
    cout << "调试模式：变量 x = " << x << endl;
#endif

// 场景2：头文件保护（防止重复包含）
#ifndef MYHEADER_H
#define MYHEADER_H
// 头文件内容...
#endif

// 场景3：定义简单的配置常量（有些人习惯用宏）
#define VERSION "1.0.0"
#define BUFFER_SIZE 1024
```

### 5.2 适合用 const 的场景（大多数情况）

```cpp
// 场景1：数学常量
const double PI = 3.14159265358979;
const double E = 2.71828182845905;

// 场景2：配置参数
const int MAX_STUDENTS = 100;
const double TAX_RATE = 0.13;
const string APP_NAME = "我的程序";

// 场景3：函数参数（防止误修改）
void printInfo(const string& name, const int age)
{
    // name 和 age 在函数内不能被修改
    cout << name << ", " << age << endl;
}

// 场景4：数组大小（C++11 推荐用 constexpr）
const int ARRAY_SIZE = 10;
int arr[ARRAY_SIZE];  // 可以用 const 常量指定数组大小
```

## 六、常见错误

### 6.1 试图修改 const 常量

```cpp
const int a = 10;
// a = 20;          // 编译错误！不能修改 const 变量
// a++;             // 编译错误！
// a = a + 1;       // 编译错误！
```

### 6.2 const 常量未初始化

```cpp
// const int x;     // 编译错误！const 变量必须在声明时初始化
const int x = 0;   // 正确
```

### 6.3 宏定义忘记加括号

```cpp
// 错误写法
#define SQUARE(x) x * x
int result = SQUARE(3 + 1);  // 替换后: 3 + 1 * 3 + 1 = 7，不是 16！

// 正确写法
#define SQUARE_CORRECT(x) ((x) * (x))
int result2 = SQUARE_CORRECT(3 + 1);  // 替换后: ((3 + 1) * (3 + 1)) = 16
```

### 6.4 宏定义末尾加分号

```cpp
// 错误写法
#define MAX 100;        // 不要加分号！
int a = MAX;            // 替换后变成: int a = 100;; （多了一个分号）
// 虽然可能不报错，但在某些场景（如 if 条件中）会导致逻辑错误

// 正确写法
#define MAX 100         // 不加分号
```

## 七、C++11 新增的 constexpr

```cpp
// constexpr 是"编译期常量"，比 const 更严格
constexpr int MAX = 100;         // 必须在编译时就能确定值
constexpr double PI = 3.14159;   // 必须在编译时就能确定值

// const 和 constexpr 的区别：
const int x = getSomeValue();    // 可以，运行时确定值也行
// constexpr int y = getSomeValue(); // 错误！constexpr 必须编译时确定

// constexpr 函数：编译时就能计算出结果
constexpr int factorial(int n)
{
    return (n <= 1) ? 1 : n * factorial(n - 1);
}

constexpr int result = factorial(5);  // 编译时就算好了，等于 120
```

## 八、综合示例

```cpp
#include <iostream>
#include <string>
using namespace std;

// 宏常量定义
#define APP_VERSION "2.0.0"
#define MAX_USERS 1000

// const 常量定义
const double PI = 3.14159265358979;
const string APP_NAME = "学生成绩管理系统";
const int PASS_SCORE = 60;

int main()
{
    cout << "===== " << APP_NAME << " v" << APP_VERSION << " =====" << endl;

    // 使用常量计算圆的面积
    double radius = 10.0;
    double area = PI * radius * radius;
    cout << "半径为 " << radius << " 的圆面积: " << area << endl;

    // 使用常量判断是否及格
    int score = 75;
    if (score >= PASS_SCORE)
    {
        cout << "成绩 " << score << " 分: 及格" << endl;
    }
    else
    {
        cout << "成绩 " << score << " 分: 不及格" << endl;
    }

    // 使用宏常量
    int userCount = 500;
    if (userCount < MAX_USERS)
    {
        cout << "还可以注册 " << MAX_USERS - userCount << " 个用户" << endl;
    }

    return 0;
}
```

## 九、总结

| 要点 | 说明 |
|------|------|
| `#define` | 文本替换，无类型检查，无作用域，不推荐用于定义常量 |
| `const` | 有类型检查，有作用域，推荐用于定义常量 |
| `constexpr` | C++11 编译期常量，最严格，推荐在编译期常量场景使用 |
| 原则 | 能用 `const` 就不用 `#define`，宏主要用于条件编译和头文件保护 |
