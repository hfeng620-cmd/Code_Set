# 运算符重载概述

## 一、什么是运算符重载？

运算符重载就是**给已有的运算符赋予新的功能**，让它们能够用于自定义类型（类、结构体）。

在 C++ 中，运算符（如 `+`、`-`、`==` 等）本来只能用于基本数据类型（int、double 等）。
通过运算符重载，我们可以让这些运算符也能作用于我们自己定义的类。

**本质：** 运算符重载其实就是**函数重载的一种特殊形式**，编译器把运算符转换成对应的函数调用。

---

## 二、为什么需要运算符重载？

### 问题演示：自定义类型不能直接用运算符

```cpp
#include <iostream>
using namespace std;

class Person {
public:
    int m_Age;
    string m_Name;

    Person(int age, string name) : m_Age(age), m_Name(name) {}
};

int main() {
    Person p1(18, "张三");
    Person p2(20, "李四");

    // Person p3 = p1 + p2;  // 错误！编译器不知道怎么对两个 Person 对象做加法

    // 如果没有运算符重载，我们只能写成函数形式：
    // Person p3 = addPerson(p1, p2);  // 不直观

    // 有了运算符重载之后，我们可以直接写：
    // Person p3 = p1 + p2;  // 直观、自然、符合人类思维

    return 0;
}
```

**总结：** 运算符重载让我们可以用更自然、更直观的方式来操作自定义类型的对象。

## 三、运算符重载的两种方式

### 方式一：成员函数重载

运算符重载函数作为**类的成员函数**，左侧操作数就是 `this` 指针指向的对象。

```cpp
#include <iostream>
#include <string>
using namespace std;

class Person {
public:
    int m_Age;
    string m_Name;

    Person() : m_Age(0), m_Name("") {}
    Person(int age, string name) : m_Age(age), m_Name(name) {}

    // 成员函数重载 + 号
    // 这里的 this 指向 p1（左操作数），参数 p 是 p2（右操作数）
    // 调用方式：p1.operator+(p2)  等价于  p1 + p2
    Person operator+(const Person &p) {
        Person temp;
        temp.m_Age = this->m_Age + p.m_Age;
        temp.m_Name = this->m_Name + "和" + p.m_Name;
        return temp;  // 返回一个新的 Person 对象
    }
};

int main() {
    Person p1(18, "张三");
    Person p2(20, "李四");

    // p1 + p2 编译器会转换为 p1.operator+(p2)
    Person p3 = p1 + p2;

    cout << "p3的姓名: " << p3.m_Name << endl;  // 输出：张三和李四
    cout << "p3的年龄: " << p3.m_Age << endl;    // 输出：38

    return 0;
}
```

**成员函数重载的要点：**
- 左操作数是 `this`，右操作数是函数参数
- 对于双目运算符（如 `+`），只需要**一个参数**（右操作数）
- 对于单目运算符（如 `++`），**不需要参数**（操作数就是 `this`）

---

### 方式二：全局函数重载

运算符重载函数作为**独立的全局函数**，所有操作数都通过参数传入。

```cpp
#include <iostream>
#include <string>
using namespace std;

class Person {
public:
    int m_Age;
    string m_Name;

    Person() : m_Age(0), m_Name("") {}
    Person(int age, string name) : m_Age(age), m_Name(name) {}
};

// 全局函数重载 + 号
// 两个操作数都作为参数传入
// 调用方式：operator+(p1, p2)  等价于  p1 + p2
Person operator+(const Person &p1, const Person &p2) {
    Person temp;
    temp.m_Age = p1.m_Age + p2.m_Age;
    temp.m_Name = p1.m_Name + "和" + p2.m_Name;
    return temp;
}

int main() {
    Person p1(18, "张三");
    Person p2(20, "李四");

    // p1 + p2 编译器会转换为 operator+(p1, p2)
    Person p3 = p1 + p2;

    cout << "p3的姓名: " << p3.m_Name << endl;  // 输出：张三和李四
    cout << "p3的年龄: " << p3.m_Age << endl;    // 输出：38

    return 0;
}
```

**全局函数重载的要点：**
- 所有操作数都是函数参数
- 对于双目运算符，需要**两个参数**
- 如果需要访问类的私有成员，需要声明为**友元函数**

---

### 两种方式的对比

| 对比项 | 成员函数重载 | 全局函数重载 |
|--------|------------|------------|
| 函数位置 | 类的内部 | 类的外部 |
| 左操作数 | this 指针 | 第一个参数 |
| 右操作数 | 函数参数 | 第二个参数 |
| 访问私有成员 | 直接访问 | 需要声明友元 |
| 使用场景 | 大多数情况 | 特殊情况（如 << 配合 cout） |

## 四、可以重载和不可以重载的运算符

### 可以重载的运算符

```cpp
/*
    可以重载的运算符（共 40 多个）：

    算术运算符：    +  -  *  /  %
    关系运算符：    ==  !=  <  >  <=  >=
    逻辑运算符：    &&  ||  !
    位运算符：      &  |  ^  ~  <<  >>
    赋值运算符：    =  +=  -=  *=  /=  %=  &=  |=  ^=  <<=  >>=
    自增自减：      ++  --
    其他运算符：    []  ()  ->  ->*  new  delete  new[]  delete[]
    逗号运算符：    ,
    条件运算符：    (注意：?: 不能重载，但这里指的是其他条件相关的)
*/
```

### 不可以重载的运算符

```cpp
/*
    不可以重载的运算符（共 5 个）：

    1. .       成员访问运算符（点运算符）
    2. .*      成员指针访问运算符
    3. ::      作用域解析运算符
    4. sizeof  大小运算符
    5. ?:      条件运算符（三目运算符）

    为什么不能重载？
    - 这些运算符在语言中有特殊的语义
    - 重载它们会导致语言行为不可预测
    - 例如：如果能重载 :: ，那类的作用域就会混乱
*/
```

## 五、运算符重载的规则

### 规则1：不能创建新的运算符

```cpp
// 错误示例：不能创建新的运算符符号
// Person operator@(const Person &p1, const Person &p2) { }  // 错误！@不是已有运算符
// Person operator**(const Person &p1, const Person &p2) { } // 错误！**不是已有运算符

// 正确：只能重载已有的运算符
Person operator+(const Person &p1, const Person &p2) { }    // 正确
Person operator-(const Person &p1, const Person &p2) { }    // 正确
```

### 规则2：不能改变运算符的操作数个数

```cpp
// + 号是双目运算符（需要两个操作数），重载时也必须是两个参数
Person operator+(const Person &p1, const Person &p2) { }    // 正确：两个参数

// 错误：不能把 + 号变成单目运算符
// Person operator+(const Person &p) { }  // 错误！+ 号不能只有一个操作数

// ++ 是单目运算符，重载时也必须是一个操作数（成员函数则无参）
// MyInteger operator++() { }       // 正确：前置++
// MyInteger operator++(int) { }    // 正确：后置++（int 是占位参数）
```

### 规则3：不能改变运算符的优先级

```cpp
/*
    重载后的运算符，优先级和原来一样，不能改变。

    例如：
    - 重载的 + 号，优先级仍然和内置的 + 号一样
    - 重载的 * 号，优先级仍然高于重载的 + 号

    这是语言规定的，不能通过重载来改变。
*/
```

### 规则4：不能改变运算符的结合性

```cpp
/*
    结合性是指当多个相同优先级的运算符连续出现时，计算的顺序。

    例如：
    - 赋值 = 是右结合：a = b = c  等价于  a = (b = c)
    - 加法 + 是左结合：a + b + c  等价于  (a + b) + c

    重载后，结合性不变。
*/
```

### 规则5：至少有一个操作数是自定义类型

```cpp
// 错误：不能重载两个基本类型之间的运算符
// int operator+(int a, int b) { return a + b; }  // 错误！两个操作数都是基本类型

// 正确：至少有一个操作数是自定义类型
class MyInt {
public:
    int value;
    MyInt(int v) : value(v) {}

    // 正确：左操作数是自定义类型
    MyInt operator+(int b) {
        return MyInt(this->value + b);
    }
};
```

## 六、完整的运算符重载示例

### 示例1：完整的加号重载

```cpp
#include <iostream>
#include <string>
using namespace std;

class Point {
public:
    int m_X;  // x 坐标
    int m_Y;  // y 坐标

    // 构造函数
    Point() : m_X(0), m_Y(0) {}
    Point(int x, int y) : m_X(x), m_Y(y) {}

    // 成员函数重载 + 号
    // this 指向左操作数，p 是右操作数
    Point operator+(const Point &p) {
        Point temp;
        temp.m_X = this->m_X + p.m_X;
        temp.m_Y = this->m_Y + p.m_Y;
        return temp;
    }

    // 成员函数重载 - 号
    Point operator-(const Point &p) {
        Point temp;
        temp.m_X = this->m_X - p.m_X;
        temp.m_Y = this->m_Y - p.m_Y;
        return temp;
    }

    // 显示坐标
    void showPoint() {
        cout << "(" << m_X << ", " << m_Y << ")" << endl;
    }
};

int main() {
    Point p1(10, 20);
    Point p2(30, 40);

    // 使用重载的 + 号
    Point p3 = p1 + p2;  // 等价于 p1.operator+(p2)
    cout << "p1 + p2 = ";
    p3.showPoint();        // 输出：(40, 60)

    // 使用重载的 - 号
    Point p4 = p2 - p1;  // 等价于 p2.operator-(p1)
    cout << "p2 - p1 = ";
    p4.showPoint();        // 输出：(20, 20)

    // 也可以和临时对象运算
    Point p5 = p1 + Point(5, 5);
    cout << "p1 + (5,5) = ";
    p5.showPoint();        // 输出：(15, 25)

    return 0;
}
```

### 示例2：全局函数重载

```cpp
#include <iostream>
#include <string>
using namespace std;

class Person {
    // 声明友元函数，让全局函数可以访问私有成员
    friend ostream& operator<<(ostream &cout, const Person &p);

public:
    string m_Name;
    int m_Age;

    Person(string name, int age) : m_Name(name), m_Age(age) {}
};

// 全局函数重载 << 运算符，配合 cout 使用
ostream& operator<<(ostream &cout, const Person &p) {
    cout << "姓名: " << p.m_Name << ", 年龄: " << p.m_Age;
    return cout;  // 返回 cout 引用，支持链式输出
}

int main() {
    Person p("张三", 18);

    // 使用重载的 << 运算符
    cout << p << endl;  // 输出：姓名: 张三, 年龄: 18

    // 链式输出（因为返回了 ostream&）
    Person p2("李四", 20);
    cout << p << " | " << p2 << endl;

    return 0;
}
```

## 七、运算符重载函数调用的两种形式

```cpp
#include <iostream>
using namespace std;

class MyInt {
public:
    int m_Value;

    MyInt(int v = 0) : m_Value(v) {}

    // 重载 + 号（成员函数形式）
    MyInt operator+(const MyInt &other) {
        return MyInt(this->m_Value + other.m_Value);
    }

    // 显示值
    void show() {
        cout << "值: " << m_Value << endl;
    }
};

// 全局函数重载 + 号（如果同时有成员函数版本，会冲突，这里注释掉）
// MyInt operator+(const MyInt &a, const MyInt &b) {
//     return MyInt(a.m_Value + b.m_Value);
// }

int main() {
    MyInt a(10);
    MyInt b(20);

    // 方式1：直接用运算符（推荐，直观）
    MyInt c = a + b;
    c.show();  // 输出：30

    // 方式2：显式调用成员函数（不推荐，但要理解原理）
    MyInt d = a.operator+(b);
    d.show();  // 输出：30

    // 方式3：显式调用全局函数（如果用全局函数重载的话）
    // MyInt e = operator+(a, b);

    // 方式4：和基本类型运算（需要有对应的重载或构造函数）
    MyInt f = a + 100;  // 100 会通过构造函数转为 MyInt(100)
    f.show();  // 输出：110

    return 0;
}
```

## 八、注意事项

1. **运算符重载不会改变运算符原有的功能**：对于基本类型，运算符的行为不受影响。

2. **不要过度使用运算符重载**：只在语义清晰、符合直觉的情况下使用。例如，`+` 用于"加法"相关的操作是合理的，但用于"减法"就违背直觉了。

3. **成员函数重载 vs 全局函数重载的选择**：
   - 一般优先使用成员函数重载
   - 当左操作数不是当前类的对象时（如 `cout << obj`），必须使用全局函数重载

4. **返回值类型很重要**：
   - 如果需要链式调用（如 `a + b + c`），返回对象本身或新对象
   - 如果需要修改原对象，考虑返回引用

5. **const 的使用**：如果不想修改操作数，参数用 `const` 修饰。

## 九、常见错误

```cpp
// 错误1：忘记了返回值
class A {
public:
    int val;
    A(int v) : val(v) {}

    // 错误：忘记返回值
    // void operator+(const A &other) {
    //     val + other.val;  // 计算了但没返回
    // }

    // 正确：
    A operator+(const A &other) {
        return A(val + other.val);
    }
};

// 错误2：参数没有用引用传递（不必要的拷贝）
class B {
public:
    int val;
    B(int v) : val(v) {}

    // 不推荐：值传递会拷贝整个对象
    // B operator+(B other) { ... }

    // 推荐：引用传递，加 const 防止意外修改
    B operator+(const B &other) {
        return B(val + other.val);
    }
};

// 错误3：试图改变运算符的操作数个数
class C {
public:
    int val;
    C(int v) : val(v) {}

    // 错误：+ 号需要两个操作数，不能只写一个
    // C operator+() { return C(val); }

    // 正确：
    C operator+(const C &other) {
        return C(val + other.val);
    }
};
```

## 十、总结

| 概念 | 说明 |
|------|------|
| 运算符重载本质 | 函数重载的特殊形式 |
| 成员函数重载 | 左操作数是 this，右操作数是参数 |
| 全局函数重载 | 所有操作数都是参数 |
| 不能创建新运算符 | 只能重载已有的运算符 |
| 不能改变操作数个数 | + 还是需要两个操作数 |
| 不能改变优先级和结合性 | 重载后和原来一样 |
| 至少一个自定义类型 | 不能重载两个基本类型之间的运算符 |

> **记住：** 运算符重载的目的是让代码更直观、更易读。如果重载后反而让代码更难理解，那就不要重载。
