# 68 - 多态案例与底层原理

## 一、案例1：计算器类

### 1.1 普通写法（不用多态）

```cpp
#include <iostream>
using namespace std;

// 普通计算器：一个类搞定所有运算
// 问题：每增加一种运算，都要修改 getResult 函数 → 违反开闭原则
class Calculator {
public:
    int num1;
    int num2;
    string oper;   // 运算符类型

    Calculator(int a, int b, string op) : num1(a), num2(b), oper(op) {}

    int getResult() {
        if (oper == "+") {
            return num1 + num2;
        } else if (oper == "-") {
            return num1 - num2;
        } else if (oper == "*") {
            return num1 * num2;
        }
        // 如果要增加除法，就要在这里加 else if → 修改了原有代码
        // 如果要增加取余，又要加 → 又修改了原有代码
        return 0;
    }
};

int main() {
    Calculator c1(10, 5, "+");
    cout << "10 + 5 = " << c1.getResult() << endl;

    Calculator c2(10, 5, "-");
    cout << "10 - 5 = " << c2.getResult() << endl;

    return 0;
}
```

**普通写法的缺点：**
```
1. 所有运算逻辑集中在一个函数中，函数越来越长
2. 每次增加新运算，都要修改 getResult → 容易引入 bug
3. 违反"开闭原则"：对扩展开放，对修改关闭
```

### 1.2 多态写法

```cpp
#include <iostream>
using namespace std;

// ==================== 抽象基类 ====================
class AbstractCalculator {
public:
    int num1;
    int num2;

    // 纯虚函数：计算结果
    virtual int getResult() = 0;

    // 虚析构
    virtual ~AbstractCalculator() {}
};

// ==================== 加法计算器 ====================
class AddCalculator : public AbstractCalculator {
public:
    int getResult() override {
        return num1 + num2;   // 加法
    }
};

// ==================== 减法计算器 ====================
class SubCalculator : public AbstractCalculator {
public:
    int getResult() override {
        return num1 - num2;   // 减法
    }
};

// ==================== 乘法计算器 ====================
class MulCalculator : public AbstractCalculator {
public:
    int getResult() override {
        return num1 * num2;   // 乘法
    }
};

int main() {
    // 加法
    AbstractCalculator *calc = new AddCalculator();
    calc->num1 = 10;
    calc->num2 = 5;
    cout << "10 + 5 = " << calc->getResult() << endl;
    delete calc;

    // 减法
    calc = new SubCalculator();
    calc->num1 = 10;
    calc->num2 = 5;
    cout << "10 - 5 = " << calc->getResult() << endl;
    delete calc;

    // 乘法
    calc = new MulCalculator();
    calc->num1 = 10;
    calc->num2 = 5;
    cout << "10 * 5 = " << calc->getResult() << endl;
    delete calc;

    return 0;
}
```

**多态写法的优点：**
```
1. 每种运算独立一个类，职责清晰
2. 增加新运算（如除法），只需新增一个 DivCalculator 类
3. 不需要修改任何已有代码 → 符合"开闭原则"
4. 通过抽象类指针统一管理，使用方式一致
```

### 1.3 对比：开闭原则

```
开闭原则（Open-Closed Principle）：
  - 对扩展开放：可以通过新增类来扩展功能
  - 对修改关闭：不需要修改已有代码

普通写法：增加新运算 → 修改 getResult 函数 → 违反开闭原则
多态写法：增加新运算 → 新增一个子类 → 符合开闭原则
```

## 二、案例2：制作饮品

```cpp
#include <iostream>
using namespace std;

// ==================== 抽象基类：制作饮品 ====================
class AbstractDrinking {
public:
    // 纯虚函数：烧水
    virtual void boil() = 0;

    // 纯虚函数：冲泡
    virtual void brew() = 0;

    // 纯虚函数：倒入杯中
    virtual void pour() = 0;

    // 纯虚函数：加配料
    virtual void addCondiment() = 0;

    // 模板方法：定义制作饮品的流程（固定顺序）
    // 这个函数不是虚函数，定义了固定的制作流程
    void makeDrink() {
        boil();           // 第1步：烧水
        brew();           // 第2步：冲泡
        pour();           // 第3步：倒入杯中
        addCondiment();   // 第4步：加配料
    }

    // 虚析构
    virtual ~AbstractDrinking() {}
};

// ==================== 子类：制作咖啡 ====================
class Coffee : public AbstractDrinking {
public:
    void boil() override {
        cout << "  1. 烧水到 100 度" << endl;
    }

    void brew() override {
        cout << "  2. 冲泡咖啡粉" << endl;
    }

    void pour() override {
        cout << "  3. 咖啡倒入杯中" << endl;
    }

    void addCondiment() override {
        cout << "  4. 加入糖和牛奶" << endl;
    }
};

// ==================== 子类：制作茶 ====================
class Tea : public AbstractDrinking {
public:
    void boil() override {
        cout << "  1. 烧水到 85 度" << endl;
    }

    void brew() override {
        cout << "  2. 冲泡茶叶" << endl;
    }

    void pour() override {
        cout << "  3. 茶水倒入杯中" << endl;
    }

    void addCondiment() override {
        cout << "  4. 加入柠檬片" << endl;
    }
};

// ==================== 子类：制作果汁 ====================
class Juice : public AbstractDrinking {
public:
    void boil() override {
        cout << "  1. 不需要烧水" << endl;
    }

    void brew() override {
        cout << "  2. 新鲜水果榨汁" << endl;
    }

    void pour() override {
        cout << "  3. 果汁倒入杯中" << endl;
    }

    void addCondiment() override {
        cout << "  4. 加入冰块" << endl;
    }
};

// ==================== 统一接口函数 ====================
// 参数是抽象类指针，传入不同子类对象，制作流程不同
void doWork(AbstractDrinking *drink) {
    drink->makeDrink();   // 调用模板方法，执行固定的制作流程
    cout << endl;
}

int main() {
    cout << "===== 制作咖啡 =====" << endl;
    Coffee coffee;
    doWork(&coffee);

    cout << "===== 制作茶 =====" << endl;
    Tea tea;
    doWork(&tea);

    cout << "===== 制作果汁 =====" << endl;
    Juice juice;
    doWork(&juice);

    return 0;
}
```

**运行结果：**
```
===== 制作咖啡 =====
  1. 烧水到 100 度
  2. 冲泡咖啡粉
  3. 咖啡倒入杯中
  4. 加入糖和牛奶

===== 制作茶 =====
  1. 烧水到 85 度
  2. 冲泡茶叶
  3. 茶水倒入杯中
  4. 加入柠檬片

===== 制作果汁 =====
  1. 不需要烧水
  2. 新鲜水果榨汁
  3. 果汁倒入杯中
  4. 加入冰块
```

## 三、案例3：电脑组装

这个案例展示了多态在"组合"设计模式中的应用。

```cpp
#include <iostream>
#include <string>
using namespace std;

// ==================== 抽象基类：CPU ====================
class CPU {
public:
    virtual void calculate() = 0;   // 纯虚函数：计算
    virtual string getName() = 0;
    virtual ~CPU() {}
};

// ==================== 抽象基类：GPU（显卡）====================
class GPU {
public:
    virtual void display() = 0;     // 纯虚函数：显示
    virtual string getName() = 0;
    virtual ~GPU() {}
};

// ==================== 抽象基类：Memory（内存）====================
class Memory {
public:
    virtual void storage() = 0;     // 纯虚函数：存储
    virtual string getName() = 0;
    virtual ~Memory() {}
};

// ==================== 具体子类：Intel CPU ====================
class IntelCPU : public CPU {
public:
    void calculate() override {
        cout << "  Intel CPU 正在进行高速计算..." << endl;
    }
    string getName() override {
        return "Intel Core i7";
    }
};

// ==================== 具体子类：AMD CPU ====================
class AMDCPU : public CPU {
public:
    void calculate() override {
        cout << "  AMD CPU 正在进行高速计算..." << endl;
    }
    string getName() override {
        return "AMD Ryzen 7";
    }
};

// ==================== 具体子类：Nvidia 显卡 ====================
class NvidiaGPU : public GPU {
public:
    void display() override {
        cout << "  Nvidia 显卡正在渲染画面..." << endl;
    }
    string getName() override {
        return "Nvidia RTX 4070";
    }
};

// ==================== 具体子类：AMD 显卡 ====================
class AMDGPU : public GPU {
public:
    void display() override {
        cout << "  AMD 显卡正在渲染画面..." << endl;
    }
    string getName() override {
        return "AMD RX 7800";
    }
};

// ==================== 具体子类：Kingston 内存 ====================
class KingstonMemory : public Memory {
public:
    void storage() override {
        cout << "  Kingston 内存正在存取数据..." << endl;
    }
    string getName() override {
        return "Kingston 16GB DDR5";
    }
};

// ==================== 具体子类：Samsung 内存 ====================
class SamsungMemory : public Memory {
public:
    void storage() override {
        cout << "  Samsung 内存正在存取数据..." << endl;
    }
    string getName() override {
        return "Samsung 32GB DDR5";
    }
};

// ==================== Computer 类：组合各组件 ====================
class Computer {
private:
    CPU *cpu;           // CPU 指针（抽象类指针）
    GPU *gpu;           // GPU 指针
    Memory *memory;     // 内存指针

public:
    // 构造函数：通过抽象类指针接收不同厂商的组件
    Computer(CPU *cpu, GPU *gpu, Memory *memory)
        : cpu(cpu), gpu(gpu), memory(memory) {}

    // 组装电脑并测试
    void assemble() {
        cout << "===== 电脑配置 =====" << endl;
        cout << "CPU: " << cpu->getName() << endl;
        cout << "GPU: " << gpu->getName() << endl;
        cout << "内存: " << memory->getName() << endl;
        cout << "====================" << endl;
    }

    // 运行电脑（多态调用各组件的功能）
    void run() {
        cout << "\n电脑启动：" << endl;
        cpu->calculate();     // 根据实际 CPU 类型调用
        gpu->display();       // 根据实际 GPU 类型调用
        memory->storage();    // 根据实际 Memory 类型调用
    }
};

int main() {
    // 方案1：Intel 全家桶
    cout << "【方案1】" << endl;
    IntelCPU cpu1;
    NvidiaGPU gpu1;
    KingstonMemory mem1;
    Computer pc1(&cpu1, &gpu1, &mem1);
    pc1.assemble();
    pc1.run();

    cout << "\n\n";

    // 方案2：AMD 全家桶
    cout << "【方案2】" << endl;
    AMDCPU cpu2;
    AMDGPU gpu2;
    SamsungMemory mem2;
    Computer pc2(&cpu2, &gpu2, &mem2);
    pc2.assemble();
    pc2.run();

    return 0;
}
```

**运行结果：**
```
【方案1】
===== 电脑配置 =====
CPU: Intel Core i7
GPU: Nvidia RTX 4070
内存: Kingston 16GB DDR5
====================

电脑启动：
  Intel CPU 正在进行高速计算...
  Nvidia 显卡正在渲染画面...
  Kingston 内存正在存取数据...


【方案2】
===== 电脑配置 =====
CPU: AMD Ryzen 7
GPU: AMD RX 7800
内存: Samsung 32GB DDR5
====================

电脑启动：
  AMD CPU 正在进行高速计算...
  AMD 显卡正在渲染画面...
  Samsung 内存正在存取数据...
```

## 四、虚函数表（vtable）底层原理

这是理解多态"为什么能工作"的关键。虽然初学者不必深入底层，但了解原理有助于理解多态。

### 4.1 核心概念

```
当一个类中有虚函数时：
  1. 编译器会为这个类创建一个"虚函数表"（vtable）
  2. 虚函数表中存储了该类所有虚函数的地址
  3. 每个该类的对象中，都有一个隐藏的指针（vptr），指向虚函数表
```

### 4.2 图解说明

```
【Animal 类（父类）】
  virtual void speak()  →  地址: 0x1000
  virtual void eat()    →  地址: 0x1004

  Animal 的虚函数表 (vtable):
  +--------+----------+
  | speak  | 0x1000   |  → Animal::speak()
  | eat    | 0x1004   |  → Animal::eat()
  +--------+----------+

  Animal 对象 a:
  +--------+
  |  vptr  | ----→ Animal 的虚函数表
  |  其他   |
  +--------+


【Cat 类（子类，重写了 speak）】
  void speak()  →  地址: 0x2000  （子类重写的版本）
  // eat 没有重写，继承父类版本

  Cat 的虚函数表 (vtable):
  +--------+----------+
  | speak  | 0x2000   |  → Cat::speak()      ← 被替换为子类的地址！
  | eat    | 0x1004   |  → Animal::eat()      ← 没有重写，保留父类地址
  +--------+----------+

  Cat 对象 c:
  +--------+
  |  vptr  | ----→ Cat 的虚函数表
  |  其他   |
  +--------+


【Dog 类（子类，重写了 speak 和 eat）】
  void speak()  →  地址: 0x3000
  void eat()    →  地址: 0x3004

  Dog 的虚函数表 (vtable):
  +--------+----------+
  | speak  | 0x3000   |  → Dog::speak()
  | eat    | 0x3004   |  → Dog::eat()
  +--------+----------+

  Dog 对象 d:
  +--------+
  |  vptr  | ----→ Dog 的虚函数表
  |  其他   |
  +--------+
```

### 4.3 调用过程

```cpp
Animal *p = new Cat();
p->speak();
```

```
调用 p->speak() 时，程序做了什么：

步骤1: 通过 p 找到 Cat 对象
步骤2: 通过 Cat 对象的 vptr 找到 Cat 的虚函数表
步骤3: 在虚函数表中找到 speak 对应的地址（0x2000 → Cat::speak）
步骤4: 调用 Cat::speak()

这就是"动态绑定"（晚绑定）的过程：
  - 在编译时，编译器不知道 p 指向的是 Cat 还是 Dog
  - 在运行时，通过 vptr → vtable → 实际函数地址，找到正确的函数
```

### 4.4 验证虚函数表的存在

```cpp
#include <iostream>
using namespace std;

class Base {
public:
    int data;
    virtual void func1() { cout << "Base::func1" << endl; }
    virtual void func2() { cout << "Base::func2" << endl; }
};

class Derived : public Base {
public:
    void func1() override { cout << "Derived::func1" << endl; }
    // func2 没有重写
};

int main() {
    Base b;
    Derived d;

    // 有虚函数的类，对象大小会增加一个指针的大小（vptr）
    cout << "Base 大小: " << sizeof(b) << endl;
    // 输出：16（int 4字节 + 对齐 4字节 + vptr 8字节，64位系统）

    cout << "Derived 大小: " << sizeof(d) << endl;
    // 输出：16（同上）

    // 没有虚函数的类
    struct NoVirtual {
        int data;
    };
    NoVirtual nv;
    cout << "NoVirtual 大小: " << sizeof(nv) << endl;
    // 输出：4（只有 int 4字节，没有 vptr）

    return 0;
}
```

### 4.5 多态调用的汇编级过程（简化版）

```
// 普通函数调用（非虚函数）
p->normalFunc();
→ 直接跳转到函数地址（编译时确定）

// 虚函数调用
p->virtualFunc();
→ mov rax, [p]              // 1. 取出对象的 vptr
→ mov rax, [rax + offset]   // 2. 从虚函数表中取出函数地址
→ call rax                  // 3. 调用该函数
```

### 4.6 用代码直观感受虚函数表

```cpp
#include <iostream>
using namespace std;

class Animal {
public:
    virtual void speak() { cout << "Animal::speak" << endl; }
    virtual void eat()   { cout << "Animal::eat" << endl; }
    virtual ~Animal() {}
};

class Cat : public Animal {
public:
    void speak() override { cout << "Cat::speak" << endl; }
    // eat 没有重写
};

class Dog : public Animal {
public:
    void speak() override { cout << "Dog::speak" << endl; }
    void eat() override   { cout << "Dog::eat" << endl; }
};

int main() {
    Animal *p;

    // 指向 Cat 对象
    Cat cat;
    p = &cat;
    p->speak();   // 输出：Cat::speak   （通过 Cat 的虚函数表找到 Cat::speak）
    p->eat();     // 输出：Animal::eat  （Cat 没有重写 eat，虚函数表中是 Animal::eat）

    cout << "---" << endl;

    // 指向 Dog 对象
    Dog dog;
    p = &dog;
    p->speak();   // 输出：Dog::speak   （通过 Dog 的虚函数表找到 Dog::speak）
    p->eat();     // 输出：Dog::eat     （Dog 重写了 eat，虚函数表中是 Dog::eat）

    return 0;
}
```

## 五、虚函数表原理总结

```
1. 有虚函数的类，编译器会为其创建虚函数表（vtable）
2. 每个对象有一个隐藏的 vptr 指针，指向所属类的虚函数表
3. 子类重写虚函数时，子类虚函数表中对应的函数地址被替换为子类版本
4. 没有重写的虚函数，子类虚函数表中保留父类版本的地址
5. 通过父类指针调用虚函数时：
   → 通过 vptr 找到虚函数表
   → 从虚函数表中取出函数地址
   → 调用该函数
6. 这个过程在运行时发生，所以叫"动态绑定"（晚绑定）

性能开销：
  - 每个对象多一个 vptr 指针（8字节，64位系统）
  - 虚函数调用多一次间接寻址（查表）
  - 通常可以忽略不计
```

## 六、常见错误与注意事项

### 错误1：混淆多态和普通函数调用

```cpp
class Animal {
public:
    void speak() { cout << "Animal" << endl; }  // 非虚函数
};

class Cat : public Animal {
public:
    void speak() { cout << "Cat" << endl; }
};

Animal *p = new Cat();
p->speak();   // 输出：Animal（不是 Cat！因为 speak 不是虚函数）
delete p;
```

### 错误2：以为子类不重写也能多态

```cpp
class Animal {
public:
    virtual void speak() { cout << "Animal" << endl; }
};

class Cat : public Animal {
    // 没有重写 speak()
};

Animal *p = new Cat();
p->speak();   // 输出：Animal（Cat 没有重写，调用父类版本）
delete p;
```

### 错误3：在构造/析构函数中调用虚函数

```cpp
class Base {
public:
    Base() {
        // func();  // 不会触发多态！调用的是 Base::func()
        // 原因：构造子类时，先构造父类，此时子类还没创建完
    }
    virtual void func() { cout << "Base" << endl; }
};
```

## 七、小结

```
案例总结：
  1. 计算器：用多态替代 if-else，符合开闭原则
  2. 制作饮品：模板方法模式，固定流程 + 多态实现各步骤
  3. 电脑组装：组合模式，通过抽象类指针组装不同厂商的组件

底层原理：
  1. 虚函数表（vtable）：存储虚函数地址的表
  2. 虚函数表指针（vptr）：每个对象中的隐藏指针，指向虚函数表
  3. 动态绑定：运行时通过 vptr → vtable → 函数地址 来调用虚函数
  4. 性能开销：多一个指针 + 一次查表，通常可忽略

设计原则：
  - 开闭原则：对扩展开放，对修改关闭
  - 依赖倒置：依赖抽象（抽象类），不依赖具体实现（具体类）
```
