# 67 - 虚析构函数详解

## 一、问题的引入：为什么需要虚析构？

先看一个**有问题**的代码：

```cpp
#include <iostream>
using namespace std;

class Animal {
public:
    Animal() {
        cout << "Animal 构造函数调用" << endl;
    }

    ~Animal() {   // 注意：不是虚析构！
        cout << "Animal 析构函数调用" << endl;
    }
};

class Cat : public Animal {
public:
    char *name;   // 用 new 分配的内存

    Cat(const char *n) {
        cout << "Cat 构造函数调用" << endl;
        // 在堆上分配内存存储名字
        name = new char[strlen(n) + 1];
        strcpy(name, n);
    }

    ~Cat() {
        cout << "Cat 析构函数调用" << endl;
        // 释放自己 new 的内存
        if (name != nullptr) {
            delete[] name;
            name = nullptr;
        }
    }
};

int main() {
    // 用父类指针指向子类对象
    Animal *animal = new Cat("Tom");

    // 使用完毕，释放内存
    delete animal;
    // 问题来了！只调用了 Animal 的析构函数，没有调用 Cat 的析构函数！
    // Cat 中 new 的 name 没有被释放 → 内存泄漏！

    return 0;
}
```

**运行结果：**
```
Animal 构造函数调用
Cat 构造函数调用
Animal 析构函数调用    ← Cat 的析构没有被调用！
```

**问题分析：**
```
1. delete animal 时，animal 的类型是 Animal*（父类指针）
2. 析构函数不是虚函数，所以编译器根据指针类型（Animal*）决定调用 Animal 的析构
3. Cat 的析构函数没有被调用
4. Cat 中 new char[] 分配的内存没有被释放 → 内存泄漏！
```

## 二、虚析构函数的解决方案

### 2.1 语法

```cpp
class Animal {
public:
    // 虚析构函数：在析构函数前加 virtual
    virtual ~Animal() {
        cout << "Animal 析构函数调用" << endl;
    }
};
```

### 2.2 完整示例

```cpp
#include <iostream>
#include <cstring>
using namespace std;

class Animal {
public:
    Animal() {
        cout << "Animal 构造函数调用" << endl;
    }

    // 加上 virtual，变成虚析构函数
    virtual ~Animal() {
        cout << "Animal 析构函数调用" << endl;
    }

    virtual void speak() {
        cout << "动物在叫" << endl;
    }
};

class Cat : public Animal {
public:
    char *name;

    Cat(const char *n) {
        cout << "Cat 构造函数调用" << endl;
        name = new char[strlen(n) + 1];
        strcpy(name, n);
    }

    ~Cat() {
        cout << "Cat 析构函数调用" << endl;
        if (name != nullptr) {
            delete[] name;
            name = nullptr;
        }
    }

    void speak() override {
        cout << name << "说：喵喵喵" << endl;
    }
};

int main() {
    Animal *animal = new Cat("Tom");
    animal->speak();   // 多态调用：输出 Tom说：喵喵喵

    delete animal;
    // 现在 Cat 的析构函数也会被调用了！
    // 内存正确释放，不会泄漏

    return 0;
}
```

**运行结果：**
```
Animal 构造函数调用
Cat 构造函数调用
Tom说：喵喵喵
Cat 析构函数调用      ← Cat 的析构被正确调用了！
Animal 析构函数调用    ← 然后调用父类的析构
```

### 2.3 前后对比

```
没有虚析构时：
  delete animal → 只调用 Animal::~Animal() → Cat 的内存泄漏

有虚析构时：
  delete animal → 先调用 Cat::~Cat() → 再调用 Animal::~Animal() → 内存正确释放
```

## 三、什么时候需要虚析构？

> **规则：当基类指针指向子类对象，并且通过基类指针 delete 时，基类的析构函数必须是虚函数。**

```cpp
// 情况1：需要虚析构
Animal *p = new Cat("Tom");
delete p;            // 通过父类指针 delete → 需要虚析构

// 情况2：不需要虚析构（但也加上不会有坏处）
Cat cat("Tom");
// cat 离开作用域时自动销毁 → 编译器知道 cat 是 Cat 类型 → 会正确调用 Cat 析构

// 情况3：不需要虚析构
Cat *p = new Cat("Tom");
delete p;            // 通过子类指针 delete → 编译器知道对象类型 → 正确调用析构
```

**简单记忆：只要类可能被继承，并且可能通过基类指针删除子类对象，就把析构函数写成虚函数。**

## 四、纯虚析构函数

### 4.1 语法

```cpp
class Animal {
public:
    // 纯虚析构函数
    virtual ~Animal() = 0;
};

// 纯虚析构函数必须在类外提供实现！
// 因为子类析构时需要调用父类析构，所以父类析构必须有函数体
Animal::~Animal() {
    cout << "Animal 纯虚析构函数调用" << endl;
}
```

### 4.2 完整示例

```cpp
#include <iostream>
#include <cstring>
using namespace std;

// ==================== 抽象类：包含纯虚析构 ====================
class Animal {
public:
    Animal() {
        cout << "Animal 构造函数调用" << endl;
    }

    // 纯虚析构函数：类内声明
    virtual ~Animal() = 0;

    // 普通纯虚函数
    virtual void speak() = 0;
};

// 纯虚析构函数：类外实现
// 这是必须的！因为子类析构时会自动调用父类析构
Animal::~Animal() {
    cout << "Animal 纯虚析构函数调用" << endl;
}

// ==================== 子类：Cat ====================
class Cat : public Animal {
public:
    char *name;

    Cat(const char *n) {
        cout << "Cat 构造函数调用" << endl;
        name = new char[strlen(n) + 1];
        strcpy(name, n);
    }

    ~Cat() {
        cout << "Cat 析构函数调用" << endl;
        if (name != nullptr) {
            delete[] name;
            name = nullptr;
        }
    }

    void speak() override {
        cout << name << "说：喵喵喵" << endl;
    }
};

int main() {
    // Animal a;              // 错误！包含纯虚析构的类是抽象类，不能创建对象

    Animal *p = new Cat("Tom");
    p->speak();
    delete p;
    // 输出：
    // Animal 构造函数调用
    // Cat 构造函数调用
    // Tom说：喵喵喵
    // Cat 析构函数调用
    // Animal 纯虚析构函数调用

    return 0;
}
```

## 五、虚析构 vs 纯虚析构 的区别

| 对比项 | 虚析构函数 | 纯虚析构函数 |
|--------|-----------|-------------|
| **语法** | `virtual ~A() { ... }` | `virtual ~A() = 0;`（类外实现） |
| **类外实现** | 不需要（函数体在类内） | 必须在类外提供实现 |
| **所在类** | 普通类或抽象类 | 只能在抽象类中 |
| **能否实例化** | 可以（如果没有其他纯虚函数） | 不能（该类变成抽象类） |
| **使用场景** | 普通基类 | 想让基类成为抽象类，但没有其他纯虚函数时 |

```cpp
// 虚析构函数示例
class Base1 {
public:
    virtual ~Base1() {
        cout << "Base1 虚析构" << endl;
    }
};
// Base1 不是抽象类，可以创建对象
// Base1 b;  // 正确

// 纯虚析构函数示例
class Base2 {
public:
    virtual ~Base2() = 0;
};
Base2::~Base2() {
    cout << "Base2 纯虚析构" << endl;
}
// Base2 是抽象类，不能创建对象
// Base2 b;  // 错误！
```

## 六、完整综合示例

```cpp
#include <iostream>
#include <string>
using namespace std;

// ==================== 抽象基类：Animal ====================
class Animal {
protected:
    string name;

public:
    Animal(string name) : name(name) {
        cout << "[Animal] " << name << " 被创建" << endl;
    }

    // 虚析构函数
    // 确保通过父类指针 delete 时，子类析构也会被调用
    virtual ~Animal() {
        cout << "[Animal] " << name << " 被销毁" << endl;
    }

    // 纯虚函数
    virtual void speak() = 0;
    virtual string getType() = 0;
};

// ==================== 子类：Cat ====================
class Cat : public Animal {
private:
    char *nickname;   // 动态分配的内存

public:
    Cat(string name, const char *nick) : Animal(name) {
        cout << "[Cat] 被创建" << endl;
        nickname = new char[strlen(nick) + 1];
        strcpy(nickname, nick);
    }

    // Cat 的析构函数：释放动态分配的内存
    ~Cat() {
        cout << "[Cat] " << nickname << " 被销毁" << endl;
        if (nickname != nullptr) {
            delete[] nickname;
            nickname = nullptr;
        }
    }

    void speak() override {
        cout << nickname << "说：喵喵喵" << endl;
    }

    string getType() override {
        return "猫";
    }
};

// ==================== 子类：Dog ====================
class Dog : public Animal {
private:
    char *nickname;

public:
    Dog(string name, const char *nick) : Animal(name) {
        cout << "[Dog] 被创建" << endl;
        nickname = new char[strlen(nick) + 1];
        strcpy(nickname, nick);
    }

    ~Dog() {
        cout << "[Dog] " << nickname << " 被销毁" << endl;
        if (nickname != nullptr) {
            delete[] nickname;
            nickname = nullptr;
        }
    }

    void speak() override {
        cout << nickname << "说：汪汪汪" << endl;
    }

    string getType() override {
        return "狗";
    }
};

// ==================== 测试函数 ====================
void test() {
    cout << "=== 创建对象 ===" << endl;
    Animal *animals[] = {
        new Cat("猫1号", "Tom"),
        new Dog("狗1号", "Buddy"),
        new Cat("猫2号", "Kitty")
    };

    cout << "\n=== 调用功能 ===" << endl;
    for (int i = 0; i < 3; i++) {
        cout << animals[i]->getType() << ": ";
        animals[i]->speak();
    }

    cout << "\n=== 释放内存 ===" << endl;
    for (int i = 0; i < 3; i++) {
        delete animals[i];   // 因为有虚析构，会正确调用子类析构
        cout << "---" << endl;
    }
}

int main() {
    test();
    return 0;
}
```

**运行结果：**
```
=== 创建对象 ===
[Animal] 猫1号 被创建
[Cat] 被创建
[Animal] 狗1号 被创建
[Dog] 被创建
[Animal] 猫2号 被创建
[Cat] 被创建

=== 调用功能 ===
猫: Tom说：喵喵喵
狗: Buddy说：汪汪汪
猫: Kitty说：喵喵喵

=== 释放内存 ===
[Cat] Tom 被销毁        ← 子类析构被正确调用！
[Animal] 猫1号 被销毁
---
[Dog] Buddy 被销毁
[Animal] 狗1号 被销毁
---
[Cat] Kitty 被销毁
[Animal] 猫2号 被销毁
---
```

## 七、常见错误与注意事项

### 错误1：纯虚析构忘记在类外提供实现

```cpp
class Animal {
public:
    virtual ~Animal() = 0;   // 声明了纯虚析构
};

// 忘记写 Animal::~Animal() { ... }
// 链接错误！子类析构时调用 Animal::~Animal() 找不到实现
```

**正确写法：**
```cpp
class Animal {
public:
    virtual ~Animal() = 0;
};

// 必须在类外提供实现
Animal::~Animal() {
    // 通常可以为空，但必须存在
}
```

### 错误2：只有析构是纯虚函数的类忘记类外实现

```cpp
class Interface {
public:
    virtual void doSomething() = 0;
    virtual void doOther() = 0;
    // 这些纯虚函数不需要在类外实现，因为子类会重写它们
    // 但纯虚析构必须在类外实现，因为子类析构时编译器会生成调用父类析构的代码
};
```

### 错误3：以为虚析构会导致无限递归

```cpp
class Animal {
public:
    virtual ~Animal() { ... }  // 虚析构
};

class Cat : public Animal {
public:
    ~Cat() { ... }             // 子类析构
    // 不需要手动调用父类析构
    // 编译器会自动在 Cat 析构函数末尾调用 Animal 的析构函数
    // 不会导致无限递归
};
```

### 注意事项总结

```
1. 基类指针指向子类对象并 delete 时，基类析构必须是虚函数
2. 虚析构语法：virtual ~类名() { ... }
3. 纯虚析构语法：virtual ~类名() = 0;（类外必须提供实现）
4. 纯虚析构使类成为抽象类
5. 即使类没有其他纯虚函数，只靠纯虚析构也能成为抽象类
6. 实际开发中，如果一个类设计为基类，通常将析构函数声明为虚函数
7. 虚析构有一点点性能开销（虚函数表），但为了安全通常值得
8. 如果确定不会通过基类指针 delete，可以不加虚析构
```

## 八、小结

```
虚析构函数：
  - 语法：virtual ~类名() { ... }
  - 作用：通过基类指针 delete 时，确保子类析构也被调用
  - 使用场景：基类指针指向子类对象并 delete

纯虚析构函数：
  - 语法：virtual ~类名() = 0;（类外实现）
  - 使类成为抽象类
  - 使用场景：想让类成为抽象类，但没有其他纯虚函数时

核心规则：
  只要类可能被继承 → 析构函数写成虚函数
  有纯虚析构 → 必须在类外提供实现
```
