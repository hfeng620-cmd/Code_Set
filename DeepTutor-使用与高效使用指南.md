# DeepTutor 使用与高效使用指南

这份文档是给你自己看的 **DeepTutor 本地使用指南**，包括：
- 怎么再次打开 DeepTutor
- 怎么配置 DeepSeek API
- 页面里每个主要功能怎么用
- 怎样更高效地使用
- 常见问题怎么排查

---

## 2. 以后怎么快速打开 DeepTutor

我已经帮你准备好了一个快捷启动命令：

```bash
deeptutor-local
```

你以后只需要在终端输入：

```bash
deeptutor-local
```

它会：
- 检查 DeepTutor 是否已经在运行
- 如果没运行，就自动启动
- 自动尝试打开浏览器

如果你想手动启动，也可以用原始命令：

```bash
cd /home/tian/Projects/DeepTutor
source .venv/bin/activate
env -u http_proxy -u https_proxy -u all_proxy -u HTTP_PROXY -u HTTPS_PROXY -u ALL_PROXY \
  python scripts/start_web.py
```

## 3. 第一次进入后最先做什么

打开页面后，建议你先做这 3 件事：

1. 进入左下角 **Settings**
2. 配置 **LLM（大模型）**
3. 再回到 **Chat** 页面测试一句话

推荐最小测试：

```text
你好，请你简单介绍一下你能帮我做什么。
```

如果能正常回复，就说明基础功能通了。

---

## 4. 如何配置 DeepSeek API

在 **Settings** 里，把 LLM 相关内容设置为：

- Provider / Binding：`deepseek`
- Model：`deepseek-chat`
- API Key：你的 DeepSeek key
- Host / Base URL：`https://api.deepseek.com`

### 建议
如果你只是想先把聊天跑通，先把 **LLM 配好** 就够了。

---

## 5. Search 要不要配 DuckDuckGo

不一定。

你可以这样理解：
- **DuckDuckGo**：最省事，不要 key
- **Serper**：相当于用 Google 搜索结果，但要 `SERPER_API_KEY`
- 其他还支持：`brave`、`tavily`、`jina`、`searxng`、`perplexity`

### 最省事方案
如果你主要是先用 DeepSeek 聊天：
- Search 可以先留空或用 `duckduckgo`

### 如果你想用 Google 搜索结果
可以配置：

```env
SEARCH_PROVIDER=serper
SEARCH_API_KEY=你的_SERPER_API_KEY
```

## 6. 页面里这些功能分别怎么用

### 6.1 Chat
最常用。
适合：
- 提问题
- 让它解释概念
- 让它出题
- 让它做学习计划
- 让它总结一段内容

示例：

```text
请用通俗的方式给我解释傅里叶变换。
```

---

### 6.2 TutorBot
适合长期陪伴式学习。

适合：
- 长期学习一个主题
- 固定一个讲解风格
- 让它像“私人家教”一样陪你

你可以把它理解为：
**比普通 Chat 更偏长期关系和个性化**。

---

### 6.3 Co-Writer
适合写作辅助。

适合：
- 改写
- 扩写
- 润色
- 总结
- 做学习笔记

例如：
- 把课堂笔记改成更清晰的版本
- 把一段文字改成更正式/更简单的表达

---

### 6.4 Guided Learning
适合做结构化学习路线。

适合：
- 学一门新课
- 分阶段学习
- 拆学习路径

例如：

```text
帮我做一个“从零开始学线性代数”的学习路径。
```

---

### 6.5 Knowledge
这个功能很重要。

你可以上传：
- PDF
- Markdown
- txt
- 讲义 / 笔记

上传之后，DeepTutor 可以围绕这些资料回答问题。

例如：

```text
根据我上传的资料，帮我总结第三章重点。
```

如果你以后想让 DeepTutor 真正变成“围绕你的资料来教学”，这个功能要多用。

---

### 6.6 Memory
适合保存长期偏好和学习背景。

可以让它记住：
- 你在学什么
- 你的基础强弱
- 你喜欢什么风格
- 你的长期目标

例如你可以告诉它：

```text
以后请用简单、一步一步、少术语的方式给我讲。
```

## 7. 最有效率的使用方法

下面是最推荐的使用顺序。

### 方法 A：先聊天，再深入
适合日常最常用。

流程：
1. Chat 提问题
2. 如果这个主题值得深挖，再去 Guided Learning
3. 如果有资料，再去 Knowledge 上传

这是最自然的用法。

---

### 方法 B：资料驱动学习
适合你有 PDF / 讲义 / 课程资料。

流程：
1. 先把资料上传到 Knowledge
2. 再在 Chat 里提问
3. 让它根据资料出题 / 总结 / 解释

推荐提问方式：

```text
根据我上传的资料，给我讲一下这一章最重要的3个概念。
```

```text
根据资料给我出5道由浅入深的练习题。
```

---

### 方法 C：把它当长期家教
适合持续学习一个主题。

流程：
1. 先在 Chat 里告诉它你的背景
2. 指定讲解风格
3. 固定让它按这个风格解释
4. 遇到难点让它继续追问和出题

推荐开场白：

```text
我基础比较弱，以后请用通俗、一步一步、少术语的方式给我讲，并且每次讲完给我一个小例子。
```

## 8. 高效提示词模板

### 8.1 让它讲概念
```text
请用最简单的话解释这个概念，并给我一个生活中的例子。
```

### 8.2 让它像老师一样讲
```text
请像老师一样一步一步讲，不要一下子堆太多术语。
```

### 8.3 让它根据你的基础来讲
```text
我基础一般，请先讲直觉，再讲定义，最后给我一道简单题。
```

### 8.4 让它根据资料来讲
```text
根据我上传的资料，帮我总结这一章的重点，并指出最容易混淆的地方。
```

### 8.5 让它出题
```text
给我出5道从简单到稍难的题，并且不要先给答案。
```

### 8.6 让它帮你复习
```text
帮我做一个3天复习计划，每天告诉我学什么、练什么、怎么检查自己会没会。
```

## 9. 最推荐的实战用法

如果你想把它用得高效，建议你长期坚持这个套路：

### 日常学习流程
1. **Chat**：先提问，理解概念
2. **Knowledge**：上传自己的资料
3. **Chat**：基于资料继续问
4. **Guided Learning**：做系统学习路线
5. **TutorBot / Memory**：形成长期陪伴式学习

这比只拿它当普通聊天机器人更值。

## 10. 常见问题

### 10.1 页面能打开，但聊天没反应
通常是：
- DeepSeek key 没填对
- 模型名填错
- 后端没有连上 provider

先去 Settings 检查 LLM 配置。

---

### 10.2 页面能打开，但知识库相关功能不好用
通常是：
- Embedding 没配置
- 资料还没处理完成

如果你暂时只聊天，这个不影响。

---

### 10.3 打不开网址
先重新运行：

```bash
deeptutor-local
```

然后再打开：

```text
http://localhost:3782
```

---

### 10.4 重启电脑以后还能直接打开吗
不能直接打开。

因为这是本地服务，不是云端常驻网站。
重启后你需要再运行一次：

```bash
deeptutor-local
```

## 12. 对你最实用的建议

如果你想真正把 DeepTutor 用起来，而不是只“打开看看”，最建议这样做：

1. 先把 **DeepSeek API 配好**
2. 先在 **Chat** 里正常聊通
3. 再上传你的课程资料到 **Knowledge**
4. 之后所有学习都尽量围绕“你的资料 + 你的问题”来进行

这样它才会越来越像“你的私人学习系统”，而不只是普通 AI 聊天。
