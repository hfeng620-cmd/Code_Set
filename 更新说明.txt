# DeepSeek Token使用量记录器 - 更新说明

## 更新内容

根据您的要求，我已经更新了DeepSeek Token使用量记录器，支持新的表结构和人民币记账。

## 新的表结构

NocoDB中的 `DeepSeek_Usage` 表现在应该包含以下列：

| 列名 | 类型 | 描述 | 必填 |
|------|------|------|------|
| Date | Date (日期) | 使用日期 | 是 |
| Model | SingleLineText (单行文本) | 模型名称，填 v3 或 r1 | 是 |
| Tokens | Number (数字) | Token使用量，整数 | 是 |
| Cost_RMB | Decimal (小数) | 使用成本，人民币 | 是 |
| task_description | LongText (长文本) | 任务描述，可选 | 否 |

## 主要更新

### 1. 脚本更新 (`deepseek_usage_logger.py`)
- ✅ 支持新的表结构字段：Date, Model, Tokens, Cost_RMB
- ✅ 人民币成本计算（基于美元定价和汇率转换）
- ✅ 模型参数现在使用 "v3" 或 "r1"（不再是 "DeepSeek", "DeepSeek-R1" 等）
- ✅ 默认模型改为 "v3"
- ✅ 更新了数据显示逻辑，兼容新旧表结构

### 2. 成本计算逻辑
- **v3模型**：$0.000001/token ≈ ¥0.0000072/token
- **r1模型**：$0.000002/token ≈ ¥0.0000144/token
- **汇率**：1美元 = 7.2人民币
- **计算示例**：
  - 1000 tokens (v3) = ¥0.0072
  - 1000 tokens (r1) = ¥0.0144

### 3. 批处理文件更新 (`log_deepseek_usage.bat`)
- ✅ 支持可选的模型参数
- ✅ 改进的错误提示
- ✅ 更详细的用法说明

1. **表名大小写**：确保NocoDB中的表名为 `DeepSeek_Usage`

## 详细使用说明

### 第一步：准备工作

1. **确保NocoDB运行**：打开浏览器访问 http://localhost:8080，确认NocoDB正常运行

2. **获取API令牌**：
   - 登录NocoDB
   - 进入设置 → API令牌
   - 生成新令牌并复制

3. **创建DeepSeek_Usage表**：
   - 在NocoDB中新建表，命名为 "DeepSeek_Usage"
   - 添加以下列：
     * Date (日期类型，Date)
     * Model (单行文本，SingleLineText，填 v3 或 r1)
     * Tokens (数字类型，Number，整数)
     * Cost_RMB (小数类型，Decimal，用于记账)
     * task_description (长文本，LongText，可选)

### 第二步：配置API令牌

选择以下任一方法配置API令牌：

#### 方法A：环境变量（推荐）
```cmd
set NOCODB_API_TOKEN=你的API令牌
```

#### 方法B：配置文件
创建文件 `D:\VS_Code\Code_Set\nocodb_config.json`：
```json
{
  "api_token": "你的API令牌",
  "nocodb_url": "http://localhost:8080"
}
```

#### 方法C：命令行参数
每次使用时添加 `--token 你的API令牌`

### 第三步：开始使用

#### 方式1：使用批处理文件（最简单）
```cmd
cd D:\VS_Code\Code_Set
log_deepseek_usage.bat 1500 "代码审查任务"
```

#### 方式2：直接使用Python脚本
```cmd
D:\AAA_Learning_Resource\ScreenTracker\venv\Scripts\python.exe deepseek_usage_logger.py --tokens 1500 --task "代码审查"
```

#### 方式3：指定模型
```cmd
D:\AAA_Learning_Resource\ScreenTracker\venv\Scripts\python.exe deepseek_usage_logger.py --tokens 2000 --model "r1" --task "架构设计"
```

### 第四步：测试连接

```cmd
D:\AAA_Learning_Resource\ScreenTracker\venv\Scripts\python.exe deepseek_usage_logger.py --test --token 你的API令牌
```

如果显示 "✅ NocoDB连接成功!" 表示配置正确。

### 第五步：查看记录和统计

```cmd

# 查看最近5条记录
D:\AAA_Learning_Resource\ScreenTracker\venv\Scripts\python.exe deepseek_usage_logger.py --recent 5 --token 你的API令牌

# 查看所有记录（设置一个较大的数字）
D:\AAA_Learning_Resource\ScreenTracker\venv\Scripts\python.exe deepseek_usage_logger.py --recent 100 --token 你的API令牌
```

### 汇总统计功能
使用 `--recent` 参数时，脚本会自动显示：
1. **按日期统计**：每天v3和r1的使用量和成本
2. **模型分类统计**：v3和r1各自的总使用量和总成本
3. **总体统计**：总Token使用量、总成本、平均每token成本
