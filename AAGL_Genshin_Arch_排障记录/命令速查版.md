## 1. 先看 AAGL 当前配置

```bash
cat /home/tian/.var/app/moe.launcher.an-anime-game-launcher/data/anime-game-launcher/config.json
```

重点看：

- `edition` 是否是 `Global`
- `game.path.global` 是否指向正确目录
- `game.wine.prefix` 是不是 `prefix-global`
- `game.wine.selected` 当前选的是哪个 runner
- `borderless` / `virtual_desktop` / `fsr.enabled` 当前是什么状态

这一步主要排除：

- 路径写错
- runner 选错
- 前缀选错

---

## 2. 看宿主机 NVIDIA 是不是正常

```bash
nvidia-smi
prime-run vulkaninfo --summary
```

判断方法：

- 如果这两条都正常，说明宿主机显卡链大体没坏
- 如果这两条都不正常，先别碰 AAGL，先修宿主机驱动

---

## 3. 看 Flatpak 里装了哪些图形 runtime

```bash
flatpak list --runtime --columns=application,branch,origin | rg 'nvidia|GL'
```

重点看：

- 宿主机 NVIDIA 版本
- Flatpak 的 `org.freedesktop.Platform.GL.nvidia-*`
- Flatpak 的 `org.freedesktop.Platform.GL32.nvidia-*`

如果宿主机是：

```text
595.71.05
```

而 Flatpak 里只有：

```text
org.freedesktop.Platform.GL.nvidia-595-58-03
org.freedesktop.Platform.GL32.nvidia-595-58-03
```

那就说明版本没对上。

这类情况非常容易导致：

- AAGL 容器里只看到 Intel
- 或者只看到 llvmpipe
- 或者 DXVK 报找不到可用适配器

---

## 4. 如果版本不匹配，补装对应的 Flatpak NVIDIA runtime

例子：

```bash
flatpak install -y flathub-official \
  org.freedesktop.Platform.GL.nvidia-595-71-05 \
  org.freedesktop.Platform.GL32.nvidia-595-71-05
```

如果默认 `flathub` 镜像出问题，先看 remote：

```bash
flatpak remotes --show-details
flatpak remotes --columns=name,url
```

如果某个镜像 SSL 出错，就改用能访问的 remote。

---

## 5. 看 AAGL 容器里现在能不能识别 NVIDIA

可以先看容器里的 Vulkan ICD：

```bash
flatpak run --command=sh moe.launcher.an-anime-game-launcher -c 'ls /usr/share/vulkan/icd.d'
```

如果你想看环境变量：

```bash
flatpak run --command=sh moe.launcher.an-anime-game-launcher -c 'echo __NV_PRIME_RENDER_OFFLOAD=$__NV_PRIME_RENDER_OFFLOAD; echo DXVK_FILTER_DEVICE_NAME=$DXVK_FILTER_DEVICE_NAME'
```

如果容器里已经能看到：

- `nvidia_icd.json`
- 正确版本的 `nvidia-*`

说明 Flatpak 图形运行时基本到位。

## 6. 直接用 AAGL 容器环境手工启动一次原神

这步很有用，因为可以绕开界面点击，直接看 Wine / DXVK / 游戏输出。

如果 runner 是 `10.0-9`，可以这样跑：

```bash
flatpak run --command=sh moe.launcher.an-anime-game-launcher -c '
cd "/media/fangfang/Genshin Impact/Genshin Impact game" &&
GST_PLUGIN_PATH="" \
LD_LIBRARY_PATH="/home/tian/.var/app/moe.launcher.an-anime-game-launcher/data/anime-game-launcher/runners/spritz-wine-cachyos-wow64-10.0-9/lib:/home/tian/.var/app/moe.launcher.an-anime-game-launcher/data/anime-game-launcher/runners/spritz-wine-cachyos-wow64-10.0-9/lib64:/home/tian/.var/app/moe.launcher.an-anime-game-launcher/data/anime-game-launcher/runners/spritz-wine-cachyos-wow64-10.0-9/lib/wine/x86_64-unix:/home/tian/.var/app/moe.launcher.an-anime-game-launcher/data/anime-game-launcher/runners/spritz-wine-cachyos-wow64-10.0-9/lib32/wine/x86_64-unix:/home/tian/.var/app/moe.launcher.an-anime-game-launcher/data/anime-game-launcher/runners/spritz-wine-cachyos-wow64-10.0-9/lib64/wine/x86_64-unix" \
WINEARCH=win64 \
WINEFSYNC=1 \
WINEPREFIX="/home/tian/.var/app/moe.launcher.an-anime-game-launcher/data/anime-game-launcher/prefix-global" \
"/home/tian/.var/app/moe.launcher.an-anime-game-launcher/data/anime-game-launcher/runners/spritz-wine-cachyos-wow64-10.0-9/bin/wine64" \
GenshinImpact.exe -window-mode borderless
'
```

这一步怎么判断：

### 情况 1

如果输出里有：

```text
DXVK: No adapters found
Found device: Intel
Found device: llvmpipe
```

说明容器里 GPU 还是没通。

### 情况 2

如果输出里有：

```text
Found device: NVIDIA GeForce GTX 1660 Ti
Creating device: NVIDIA GeForce GTX 1660 Ti
```

说明图形链已经通了。

### 情况 3

如果输出里还有：

```text
Library WDFLDR.SYS ... needed by HoYoKProtect.sys not found
```

说明图形层过了，但 runner / 兼容层还不稳。

## 9. 更稳的启动参数建议

如果只是想先“确保能起起来”，建议先这样：

```json
"borderless": true
```

```json
"virtual_desktop": {
  "enabled": true
}
```

```json
"fsr": {
  "enabled": false
}
```

原因：

- 先避免独占全屏带来的变量
- 先避免 FSR 叠加问题
- 先把目标缩成“游戏能正常出窗口”

## 11. 前缀操作原则

### 不要一上来就删前缀

先备份：

```bash
cp -a prefix-global prefix-global.bak-时间戳
```

或者：

```bash
mv prefix-global prefix-global.bak-时间戳
```

### 为什么

因为前缀里不只是 Wine 基础结构，还有：

- 原神运行态
- 登录页缓存
- 图形缓存
- 注册表状态
- 兼容层初始化结果

删掉前缀有时会把“可用状态”一起删掉。

## 14. 最短判断流程

如果以后只想 2 分钟内判断大概问题在哪，直接按这个走：

### 1

```bash
nvidia-smi
prime-run vulkaninfo --summary
```

这两条错了，先修宿主机。

### 2

```bash
flatpak list --runtime --columns=application,branch,origin | rg 'nvidia|GL'
```

宿主机版本和 Flatpak `GL.nvidia-*` 对不上，先补 runtime。

### 3

直接在 AAGL 容器里试启动，看输出：

- 只看到 Intel / llvmpipe：还是图形链问题
- 能看到 NVIDIA，但撞 `WDFLDR`：换 runner
- 能看到 NVIDIA 且 DXVK 创建设备成功：已经进入更深层，继续看 `output_log.txt`

## 15. 一句话记忆版

遇到 “昨天还能玩，今天 AAGL 原神进不去” 这种问题，优先怀疑这三件事：

1. Flatpak 的 NVIDIA runtime 有没有跟上宿主机驱动版本。
2. AAGL 当前 runner 是不是太旧。
3. 是否误动了 AAGL 自己的 `prefix-global`。
