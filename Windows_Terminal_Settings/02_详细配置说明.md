# Windows Terminal 详细配置说明

## 一、整体结构

这套终端外观不是单靠一个软件完成的，而是下面几层叠加：

### 1. Windows Terminal
负责：

- 窗口展示
- 标签页
- 默认启动哪个 Shell
- 字体
- 背景透明
- 快捷键
- 配色方案

### 2. PowerShell 7
负责：

- 实际命令执行
- 加载配置文件
- 提供 shell 变量和函数
- 给 oh-my-posh 提供运行环境

### 3. oh-my-posh
负责：

- 左侧彩色 prompt
- 右侧状态信息
- Git 状态展示
- 执行状态和耗时显示
- 不同段的颜色和图标

### 4. Nerd Font
负责：

- 图标字体
- Powerline 分隔符
- 避免 prompt 乱码或错位

## 二、Windows Terminal 当前配置

配置文件位置：

`C:\Users\Administrator\AppData\Local\Packages\Microsoft.WindowsTerminal_8wekyb3d8bbwe\LocalState\settings.json`

### 1. 默认 Profile

当前值：

`{574e775e-4f2a-5b96-ac1e-a2962a402336}`

对应：

`PowerShell 7`

它实际执行：

`C:\Users\Administrator\AppData\Local\Programs\PowerShell\7\pwsh.exe -NoLogo`

说明：

- 打开 Windows Terminal 时默认进入 PowerShell 7
- `-NoLogo` 用来去掉 PowerShell 启动横幅，界面更干净

### 2. Profiles 默认项

当前配置：

- `bellStyle`: `none`
- `colorScheme`: `NeonNoir`
- `cursorShape`: `bar`
- `font.face`: `JetBrainsMono NFM`
- `font.size`: `12`
- `opacity`: `90`
- `padding`: `12, 10, 12, 10`
- `scrollbarState`: `hidden`
- `useAcrylic`: `true`

说明：

- `bellStyle: none` 关闭提示铃
- `colorScheme: NeonNoir` 使用自定义霓虹配色
- `cursorShape: bar` 使用竖线光标，观感更利落
- `JetBrainsMono NFM` 是 Nerd Font 字体，确保图标完整
- `opacity: 90` 让背景轻微透明
- `padding` 给终端内容留出呼吸感
- `scrollbarState: hidden` 隐藏滚动条，让界面更简洁
- `useAcrylic: true` 开启 Windows Terminal 的亚克力模糊效果

### 3. 配色方案 NeonNoir

这个方案是后来添加的自定义 scheme，主要颜色是：

- 背景：`#090B14`
- 前景：`#EAF2FF`
- 光标：`#FF5FD2`
- 蓝色：`#59A7FF`
- 青色：`#4DDCFF`
- 绿色：`#66F2B3`
- 紫色：`#A97CFF`
- 红色：`#FF6B81`
- 黄色：`#FFCB6B`

设计目的：

- 背景足够深，衬托高饱和点缀色
- 主要高亮偏蓝、青、紫、粉，形成偏赛博风的视觉
- 前景文字保持浅色，避免可读性下降

- `Ctrl+Shift+C` 复制

- `Ctrl+Shift+W` 关闭窗格

## 三、PowerShell 当前配置

配置文件位置：

`C:\Users\Administrator\Documents\PowerShell\Microsoft.PowerShell_profile.ps1`

### 1. PSReadLine 设置

当前启用了：

- 历史预测：`Set-PSReadLineOption -PredictionSource History`
- 列表显示预测：`Set-PSReadLineOption -PredictionViewStyle ListView`
- 编辑模式：`Windows`
- 禁用铃声：`BellStyle None`

说明：

- 输入命令时会基于历史记录给出提示
- `ListView` 会把候选项展开成列表
- 这部分影响输入体验，不影响配色

### 2. 按键绑定

当前设置：

- `Tab` -> `MenuComplete`
- `UpArrow` -> `HistorySearchBackward`
- `DownArrow` -> `HistorySearchForward`

说明：

- `Tab` 用菜单方式补全
- 上下键不是简单翻历史，而是按当前输入内容搜索历史

### 3. 自定义函数

当前定义：

- `ll`：`Get-ChildItem -Force`
- `la`：`Get-ChildItem -Force -Hidden`
- `grep`：递归搜索文本

说明：

这些是方便命令行使用的小别名/函数，和终端样式关系不大。

### 4. 代理同步函数

当前有一个 `Sync-ProxyEnvFromSystemProxy`，作用是：

- 从系统代理读取配置
- 自动写入 `HTTP_PROXY`
- 自动写入 `HTTPS_PROXY`
- 自动写入 `ALL_PROXY`
- 自动写入 `NO_PROXY`

说明：

这部分和外观无关，但会影响命令行里的网络访问行为。

### 5. oh-my-posh 初始化

当前初始化逻辑：

```powershell
if (Get-Command oh-my-posh -ErrorAction SilentlyContinue) {
    oh-my-posh init pwsh --config "D:\OneDrive\文档\AutoHotkey\terminal\neon-synthwave.omp.json" | Invoke-Expression
} elseif (Test-Path "C:\Users\Administrator\AppData\Local\Programs\oh-my-posh\bin\oh-my-posh.exe") {
    & "C:\Users\Administrator\AppData\Local\Programs\oh-my-posh\bin\oh-my-posh.exe" init pwsh --config "D:\OneDrive\文档\AutoHotkey\terminal\neon-synthwave.omp.json" | Invoke-Expression
}
```

说明：

- 优先用环境里的 `oh-my-posh`
- 如果命令不可用，就直接调用安装路径下的 exe
- `--config` 指定你自定义的 prompt 主题文件

### 6. TERM_PROGRAM

当前还设置了：

`$env:TERM_PROGRAM = "Windows Terminal"`

作用：

- 让一部分工具知道当前宿主终端是 Windows Terminal

## 四、oh-my-posh 当前配置

主题文件位置：

`D:\OneDrive\文档\AutoHotkey\terminal\neon-synthwave.omp.json`

### 1. 主题定位

这份主题是自定义的，风格目标是：

- 比默认主题更花哨
- 保留较高可读性
- 用高对比霓虹色突出不同信息块

### 2. 左侧 prompt 段

从左到右主要有这些段：

- `session`
  显示用户名。管理员身份时会很醒目。

- `os`
  显示操作系统图标。

- `path`
  显示当前路径，使用文件夹图标和 Powerline 连接。

- `git`
  显示分支、上游图标、工作区和暂存区状态。
  颜色会根据 Ahead/Behind 和修改状态变化。

- `executiontime`
  当命令执行超过 `500ms` 时显示耗时。

- `root`
  当前是管理员终端时显示闪电图标段。

- `status`
  显示上一个命令成功还是失败。
  成功是对勾，失败会显示叉号和退出码。

### 3. 右侧 rprompt 段

当前右侧提示包括：

- `node`
  有 Node 项目文件时显示 Node 版本段

- `python`
  有 Python 项目文件时显示 Python 或虚拟环境信息

- `shell`
  显示当前 shell 名称

- `time`
  显示当前时间

### 4. secondary 和 transient prompt

当前还定义了：

- `secondary_prompt`
  在续行输入时显示简短提示

- `transient_prompt`
  命令执行后把历史 prompt 压缩成更短版本，界面更清爽

### 5. tooltips

当前给这两个命令配置了 tooltip：

- `aws`
- `az`

说明：

输入相关命令时，oh-my-posh 可以额外显示云环境相关信息。

## 五、字体配置

当前字体：

`JetBrainsMono NFM`

来源：

- 使用 `winget` 安装 `DEVCOM.JetBrainsMonoNerdFont`

作用：

- 支持 Nerd Font 图标
- 支持 Powerline 分隔符
- 与开发场景比较匹配
- 比普通系统字体更适合终端排版

如果把字体换回普通字体，可能出现：

- 菱形分隔符错位
- 图标显示成方块
- prompt 宽度计算异常

## 六、如果要手动复刻

### 步骤 1
安装 PowerShell 7。

### 步骤 2
安装 oh-my-posh。

### 步骤 3
安装 Nerd Font，例如 JetBrains Mono Nerd Font。

### 步骤 4
在 Windows Terminal 中把字体改成 `JetBrainsMono NFM`。

### 步骤 5
把 `settings.json` 中的配色、透明度、快捷键等设置同步过去。

### 步骤 6
在 PowerShell profile 中加入 oh-my-posh 初始化代码。

### 步骤 7
把 `neon-synthwave.omp.json` 放到固定路径，并在 profile 中用 `--config` 指向它。

### 步骤 8
重启 Windows Terminal。
