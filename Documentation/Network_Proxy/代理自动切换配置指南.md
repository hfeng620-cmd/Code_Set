## 问题描述

### 现象
- **开启梯子时**：Qwen Code 和 Hermes 都能正常使用
- **关闭梯子时**：两个工具都无法连接，报错或卡住

### 根本原因

系统中设置了全局代理环境变量：

```bash
http_proxy=http://127.0.0.1:7897
https_proxy=http://127.0.0.1:7897
HTTP_PROXY=http://127.0.0.1:7897
HTTPS_PROXY=http://127.0.0.1:7897
```

这些变量通常在 `~/.zshrc` 或 `~/.bashrc` 中设置。

**问题机制**：
1. 开启梯子 → 代理服务器 `127.0.0.1:7897` 正常运行 → 请求通过代理转发 → 正常工作
2. 关闭梯子 → 代理服务器停止 → 但环境变量还在 → 所有请求仍尝试连接 `127.0.0.1:7897` → 连接失败

**额外问题**：
- `~/.zshrc` 中的 `qwen` 函数强制使用代理，即使直连也能访问的服务也会走代理
- DeepSeek 的 API 地址 `https://api.deepseek.com/v1` 在国内是可以直接访问的，不需要代理

---

## 解决方案

### 1. 删除原有的硬编码代理设置

原来 `.zshrc` 中的配置（需要删除/替换）：

```bash
# 旧配置 - 强制使用代理，删除或替换
alias qwen-http='env -u all_proxy -u ALL_PROXY HTTP_PROXY=http://127.0.0.1:7897 ...'
qwen() {
  env -u all_proxy -u ALL_PROXY \
    HTTP_PROXY=http://127.0.0.1:7897 \
    HTTPS_PROXY=http://127.0.0.1:7897 \
    http_proxy=http://127.0.0.1:7897 \
    https_proxy=http://127.0.0.1:7897 \
    /home/tian/.npm-global/bin/qwen "$@"
}
```

**新配置 - 动态代理**：

```bash
qwen() {
  # 动态使用当前代理设置
  if [ -n "$https_proxy" ]; then
    # 有代理时：清除系统代理变量，手动设置
    env -u all_proxy -u ALL_PROXY \
      HTTP_PROXY=http://127.0.0.1:7897 \
      HTTPS_PROXY=http://127.0.0.1:7897 \
      http_proxy=http://127.0.0.1:7897 \
      https_proxy=http://127.0.0.1:7897 \
      /home/tian/.npm-global/bin/qwen "$@"
  else
    # 无代理时：直连模式
    /home/tian/.npm-global/bin/qwen "$@"
  fi
}
```

### 2. 添加代理自动检测函数

在 `~/.zshrc` 末尾添加以下内容：

```bash
# ==========================================
# 代理自动检测与切换
# ==========================================
# 检测代理是否可用
_check_proxy() {
    # 检查代理端口是否可达
    if timeout 1 bash -c "echo >/dev/tcp/127.0.0.1/7897" 2>/dev/null; then
        return 0  # 代理可用
    else
        return 1  # 代理不可用
    fi
}

# 自动检测并切换代理状态
auto_proxy() {
    if _check_proxy; then
        # 代理端口可达
        export http_proxy=http://127.0.0.1:7897
        export https_proxy=http://127.0.0.1:7897
        export HTTP_PROXY=http://127.0.0.1:7897
        export HTTPS_PROXY=http://127.0.0.1:7897
        echo "🌐 代理已自动启用 (127.0.0.1:7897)"
    else
        # 代理端口不可达
        unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY
        echo "🌐 代理已自动关闭（直连模式）"
    fi
}

# 开启代理（手动）
proxy_on() {
    export http_proxy=http://127.0.0.1:7897
    export https_proxy=http://127.0.0.1:7897
    export HTTP_PROXY=http://127.0.0.1:7897
    export HTTPS_PROXY=http://127.0.0.1:7897
    echo "✓ 代理已开启 (127.0.0.1:7897)"
}

# 关闭代理（手动）
proxy_off() {
    unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY
    echo "✓ 代理已关闭（直连模式）"
}

# 查看当前代理状态
proxy_status() {
    if [ -n "$https_proxy" ]; then
        if _check_proxy; then
            echo "🌐 代理状态: 开启 -> $https_proxy ✓"
        else
            echo "🌐 代理状态: 开启但不可用 ✗（建议运行 auto_proxy）"
        fi
    else
        echo "🌐 代理状态: 关闭（直连）"
    fi
}

# 每次打开终端时自动检测
auto_proxy 2>/dev/null
```

### 3. 使配置生效

```bash
source ~/.zshrc
```

或者关闭当前终端，打开新终端即可自动生效。

## 使用方法

### 日常使用（全自动）

1. **开启梯子** → 打开新终端 → 自动启用代理 ✓
2. **关闭梯子** → 打开新终端 → 自动切换直连 ✓
3. 正常使用 `qwen`、`hermes` 等工具


## 注意事项

1. **代理端口**：如果你的代理端口不是 `7897`，需要将配置中的 `7897` 替换为实际端口
2. **其他工具**：Hermes 等其他工具会自动读取系统代理变量，不需要额外配置
3. **网络环境变化**：如果梯子地址或端口变更，更新 `~/.zshrc` 中的端口即可
4. **手动触发**：在已经打开的终端中切换梯子状态后，运行 `auto_proxy` 即可刷新代理状态
