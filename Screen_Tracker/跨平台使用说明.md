# 屏幕使用时长统计工具 - 跨平台版本使用说明

## 概述

这是一个跨平台的屏幕使用时长统计工具，支持 Windows 和 Linux (Arch Linux) 双系统。工具会自动检测用户的活动应用并记录使用时间，生成可视化报表。

## 文件说明

- `ScreenTracker_cross_platform.pyw` - 跨平台主程序
- `ScreenTracker.pyw` - 原始 Windows 版本（备份）
- `install_dependencies.py` - 依赖安装脚本
- `screen_data.db` - 使用数据数据库（Windows 和 Linux 共享）
- `icon.png` - 程序图标（可选）

## 快速开始

### 1. 安装依赖

在运行程序前，需要先安装必要的依赖：

```bash
# 运行依赖安装脚本
python install_dependencies.py
```

脚本会自动检测操作系统并安装相应的依赖包。

### 2. 运行程序

```bash
# 运行跨平台版本
python ScreenTracker_cross_platform.pyw
```

程序启动后会显示系统托盘图标（如果支持），并在后台监控屏幕使用情况。

## 操作系统特定配置

### Windows 系统

#### 依赖要求
- Python 3.6+
- pywin32 (自动安装)
- 其他 Python 包：psutil, pillow, matplotlib, pystray

#### 功能特点
- 使用 Windows API 精确获取活动窗口
- 支持注册表开机自启
- 原生系统托盘支持
- 精确的空闲时间检测

#### 常见问题
1. **系统托盘图标不显示**
   - 尝试以管理员身份运行
   - 检查防病毒软件是否阻止

2. **开机自启失败**
   - 需要注册表写入权限
   - 检查用户账户控制设置

### Linux 系统 (Arch Linux)

#### 系统工具要求
```bash
# Arch Linux 安装系统工具
sudo pacman -S xdotool xprintidle wmctrl

# 对于 GNOME 桌面环境
sudo pacman -S libappindicator-gtk3

# 对于 KDE Plasma（通常已内置支持）
```

#### Python 依赖
```bash
# 安装 Python 包
pip install psutil pillow matplotlib pystray python3-xlib
```

#### 功能特点
- 使用 xdotool 获取活动窗口
- 使用 xprintidle 检测空闲时间
- 支持 .desktop 文件开机自启
- 系统托盘支持取决于桌面环境

#### 桌面环境兼容性

**GNOME**
- 需要安装 `libappindicator-gtk3`
- 可能需要启用扩展支持
- 系统托盘支持可能有限

**KDE Plasma**
- 通常有更好的系统托盘支持
- 不需要额外配置

**XFCE**
- 良好的系统托盘支持
- 可能需要安装通知守护进程

**其他桌面环境**
- 尝试运行程序查看兼容性
- 如果不支持系统托盘，程序会在后台运行

## 功能说明

### 1. 数据记录
- 每5秒检测一次活动应用
- 10分钟无操作视为空闲，不记录时间
- 数据存储在 SQLite 数据库 `screen_data.db` 中

### 2. 系统托盘菜单
- 📊 今日报表 - 显示当天使用统计
- 📈 本周报表 - 显示本周使用统计
- 📅 本月报表 - 显示本月使用统计
- ✅ 开机自启 - 设置开机自动启动
- ❌ 取消自启 - 取消开机自动启动
- 退出程序 - 停止程序运行

### 3. 统计报表
- 环形图显示应用时间占比
- 文本列表显示详细使用时长
- 支持日报、周报、月报
- 自动美化应用名称

### 4. 跨平台数据共享
- 数据库文件格式相同
- Windows 和 Linux 数据可以互相读取
- 应用名称映射兼容两个系统

## 高级配置

### 修改检测间隔
编辑 `ScreenTracker_cross_platform.pyw` 中的配置项：
```python
INTERVAL = 5  # 检测间隔（秒）
IDLE_THRESHOLD = 600  # 空闲阈值（秒）
```

### 添加应用名称映射
在 `APP_MAP` 字典中添加新的映射：
```python
APP_MAP = {
    "your_app.exe": "应用名称",
    "your_linux_app": "应用名称",
    # ...
}
```

### 自定义图标
将自定义图标文件命名为 `icon.png` 放在程序目录下，程序会自动加载。

## 故障排除

### 常见问题

1. **程序无法启动**
   - 检查 Python 版本（需要 3.6+）
   - 运行 `install_dependencies.py` 安装依赖
   - 查看错误信息进行调试

2. **无法检测活动窗口**
   - Windows：检查 pywin32 是否安装正确
   - Linux：检查 xdotool 是否安装并可用
   - 尝试重启程序

3. **系统托盘不显示**
   - Linux：检查桌面环境兼容性
   - 尝试安装必要的系统包
   - 程序会在后台运行，按 Ctrl+C 停止

4. **报表无法显示**
   - 检查 matplotlib 是否安装
   - 确保有足够的数据（至少1分钟使用时间）
   - 检查数据库文件权限

### 调试模式
可以修改代码启用调试输出：
```python
# 在 MonitorThread.run() 方法中添加调试输出
print(f"空闲时间: {idle_time}, 活动应用: {app}")
```

## 数据迁移

### 从 Windows 迁移到 Linux
1. 将 `screen_data.db` 文件复制到 Linux 系统
2. 确保文件权限正确（可读写）
3. 运行程序，数据会自动加载

### 从 Linux 迁移到 Windows
1. 将 `screen_data.db` 文件复制到 Windows 系统
2. 运行程序，数据会自动加载

## 安全注意事项

1. **数据隐私**
   - 所有数据存储在本地
   - 不会上传到任何服务器
   - 数据库文件可以加密（需要额外配置）

2. **系统权限**
   - Windows：需要注册表写入权限（开机自启）
   - Linux：需要 X11 访问权限（活动窗口检测）
   - 程序不会请求不必要的权限

3. **防病毒软件**
   - 某些防病毒软件可能误报
   - 可以将程序目录添加到白名单
